# VIDU 来访 · 提问策略与准备

## 核心身份定位

**你不是「技术用户」去求教——你是「电商 AI Agent 落地顾问」去对齐。**

你的价值是什么：
- 你手上有客户的真实电商场景和需求
- 你正在搭建可复用的电商 AIGC 工作流
- 你是 VIDU 触达企业客户的桥梁

这意味着——你是他们想争取的人，你是来评估"这工具能不能帮我的客户解决问题"的。

---

## 三层提问框架

### L1 · 基础摸底（开场 10-15 分钟）

**目的**：摸清 VIDU 当前的真实能力边界，确认基础可用性

#### 1.1 模型能力边界

| 问题 | 为什么问 |
|------|----------|
| "目前文生视频的最高分辨率/时长是多少？支持 4K 输出吗？" | 电商商品视频需要高清细节（纹理、标签、包装文字） |
| "图生视频的一致性控制到什么程度？" | 核心场景：给商品图→生成展示视频，必须保持商品外观不变 |
| "支持哪些镜头运动控制？推拉摇移跟？" | 电商视频需要标准化运镜语言 |
| "多片段/多镜头的生成长度上限是多少？" | 一条 15-30 秒的电商视频通常需要多段拼接 |
| "视频风格控制能做到什么粒度？" | 品牌调性一致性是客户刚需 |

#### 1.2 API 技术规格与成本

| 问题 | 为什么问 |
|------|----------|
| "API 的并发上限和延迟大概是多少？" | 批量场景下（一次生成 100 条商品视频）是否可行 |
| "定价模式是怎样的？按时长/按分辨率/按 token？" | 帮助客户估算 ROI |
| "是否有异步生成 + Webhook 回调机制？" | 这是嵌入 Agent 工作流的**必须条件** |
| "输出格式有哪些？MP4 与逐帧序列帧支持情况？" | 后期剪辑/合成需求 |
| "水印策略是什么？企业版是否有去水印方案？" | 客户方的品牌合规 |

---

### L2 · 电商工作流深度融合（核心环节 20-30 分钟）

**这是全场最重要的 30 分钟。** 你的工作流方案可以直接摆出来问。

#### 2.1 批量 · 一致性 · 可控性

**推荐展示方式**：打开你的 ReactFlow 工作流界面/架构图，指着某个节点问。

**核心问题链**：

1. **"我每天可能要给 500-1000 个 SKU 生成展示视频，如何保证同一商品在不同 prompt 下视觉一致？"**
   - 追问：是否有 Seed 控制 / 参考图权重调节 / 风格锁定机制
   - 场景：一个保温杯，从「旋转展示」变成「开盖倒水演示」，杯体外观不能变

2. **"对于电商常见类目，有没有专项优化？"**
   - 服装：动态展示（衣料飘动、色彩还原）
   - 美妆：肤质细节、颜色精准度
   - 3C 数码：屏幕内容、光效、接口细节
   - 食品：质感、蒸汽、沸腾等动态效果

3. **"支持 LoRA 或类似微调吗？"**
   - 场景：某品牌有一整套视觉资产（品牌色、模特脸、特定布光），能否训练品牌专属模型
   - 训练成本、数据要求、部署方式

#### 2.2 工作流嵌入深度

**直接展示你的架构**：「我目前的工作流是 Vite+React+ReactFlow 构建的，AI Agent 编排节点——从商品数据采集 → 文案生成 → 视觉方案 → 视频生成 → 发布到电商平台。VIDU 在哪个环节嵌入最合适？」

**具体问题**：

| 问题 | 为什么问 |
|------|----------|
| "API 的异步回调机制是如何设计的？" | 工作流中视频生成是阻塞节点，需要可靠的完成通知 |
| "是否支持多 prompt 排队生成？队列管理机制？" | 批量场景的基础设施 |
| "能否与飞书多维表格联动？比如表格里一条商品记录→自动触发视频生成→结果写回表格" | 这是你目前在用的一体化方案 |
| "是否有视频资产管理的 API（存储/版本/检索/删除）？" | 素材管理的工程化需要 |
| "视频生成失败的重试和错误码体系完善吗？" | 企业级稳定性要求 |

#### 2.3 行业定制化纵深

**身份切换：以 AI 落地培训师 + GEO 顾问的角度提问**

- "你们在电商/零售行业有没有标杆客户？他们怎么用的？踩过什么坑？"
- "从你们的角度，电商客户最常提的三种需求是什么？最让你们头疼的是什么？"
- "未来 3-6 个月的 roadmap 上，电商场景的优先级如何？"
- "对于视频内容的 SEO/GEO（生成引擎优化），VIDU 有考虑吗？——比如视频帧文本的语义索引、视频摘要生成这些？"

---

### L3 · 战略合作层（收尾 10 分钟，拉高价值）

**目的**：把一次拜访变成长期合作通道

#### 3.1 企业培训合作

"我这边在做 AI 落地培训，每年接触不少企业和品牌方。VIDU 在企业侧有没有培训合作计划？——比如我可以把 VIDU 作为电商视频生成方案推荐给客户，你们提供技术支持和案例材料。"

#### 3.2 产品反馈通道

"我如果深度用 VIDU 做一批电商场景测试，你们有内测用户群/技术支持群吗？——我们是能给出高质量产品反馈的深度用户。"

#### 3.3 标杆案例共创

"如果我用 VIDU 做出一个完整的电商 AIGC 工作流 demo，你们有兴趣合作出案例吗？白皮书、联合宣讲之类的。"

## 会前准备 Checklist

### 需要准备的材料
- [ ] 你的电商工作流架构图（一页纸的 ReactFlow 图示）
- [ ] 1-2 个具体商品的需求描述（选一个品类，比如保温杯或口红）
- [ ] 你的飞书多维表格里的一条商品记录结构
- [ ] 之前做过的 LV/品牌 logo 视频案例（展示你是实际用过的用户）
- [ ] 你的业务介绍（一句话：做电商 AI 落地 + GEO 培训的顾问）

### 开场建议
**不要一上来就问价格。** 开场先介绍你的业务和客户场景，让对方知道「来的人值得认真对待」。然后从 L1 开始自然过渡到 L2。

### 收场必备动作
- 加核心联系人微信/飞书
- 问 API 文档入口（或邀请进开发者群）
- 要一份 SDK / 快速接入指南
- 确认后续是否可以做技术对接

## 如果只有 3 个问题的时间

优先级排序：

1. **"电商批量场景下，如何保证同一个 SKU 在不同 prompt 下的视觉一致性？"**（一致性）
2. **"API 的异步 Webhook 回调机制是怎样的？"**（工作流可嵌入性）
3. **"你们在电商方向最近的 roadmap 是什么？"**（合作方向判断）
