## 解决 lark-cli 沙箱授权卡点
- 根因：lark-cli 原生读取 `~/.lark-cli/`（C盘），每次操作都触发 WorkBuddy 沙箱授权
- 方案：在 workspace 内创建 `Claw/lark-cli`（bash）+ `Claw/lark-cli.bat`（batch）包装脚本
- 关键：设置 `USERPROFILE` 和 `HOME` 指向 `Claw/` 目录，使 lark-cli 读取 `Claw/.lark-cli/config.json`
- 已验证：`./lark-cli docs +fetch` 无沙箱告警，config 路径已指向 workspace 内
- 从此 lark-cli 操作不再需要手动授权
