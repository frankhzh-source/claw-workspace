## 概览：AI产业的四层结构

AI产业不是只有"大模型"。把AI理解为一条生产线：最底层是算力和数据（修路架桥），中间层是模型引擎（造发动机），再往上是工具链（造方向盘仪表盘），最顶层是落地应用（开车上路）。

| 层级 | 价值占比 | 一句话 | 核心玩家 | 小白关键词 |
|-|-|-|-|-|
| 基础层 · 算力与数据底座 | **40%** | 修路架桥 | 英伟达、AMD、华为、云厂商 | GPU、算力、数据中心 |
| 技术层 · 智能引擎 | **30%** | 造发动机 | OpenAI、Anthropic、字节、阿里 | 大模型、多模态、API |
| 中间层 · 协议与工具链 | **10%** | 造方向盘 | LangChain、Dify、向量数据库 | Agent、RAG、Prompt |
| 应用层 · 场景价值出口 | **20%** | 开车上路 | 行业解决方案商、SaaS公司 | 智能客服、AI视频 |

## 第一层：基础层 · 算力与数据底座（40%）

所有AI应用需要的"体力活"——提供计算能力（GPU芯片）、存储数据（数据中心）、运送数据（网络）。没有这一层，上面三层什么都跑不起来。

**初学者切入点**：不需要懂芯片设计。理解三件事就够了——①GPU是什么（专门做并行计算的芯片）②云服务vs本地部署的区别 ③为什么英伟达一家独大。

### 1.1 AI芯片
- **通用GPU**：英伟达 H100/B200/GB200（训练市场77%）、AMD MI300X、Intel Gaudi 3
- **国产替代**：华为昇腾910B/310P、寒武纪MLU370/290、海光DCU
- **端侧/边缘**：高通骁龙8 Elite（NPU）、苹果M4/M3、地平线征程6（车载）
- **专用ASIC**：Google TPU v5、Amazon Trainium2、阿里含光800

### 1.2 AI服务器与智算中心
浪潮信息、工业富联、阿里云灵骏、腾讯云星脉、火山引擎GPU集群

### 1.3 云服务三层结构
- **IaaS**：按需租服务器（阿里云ECS、AWS EC2）— 技术团队
- **PaaS/MaaS**：直接调模型API（阿里百炼、火山方舟）— 开发者
- **SaaS**：开箱即用（钉钉AI、飞书智能伙伴）— 普通用户

### 1.4 数据基础设施
数据采集→标注→清洗→向量存储（FAISS/Milvus/Chroma）
互联：InfiniBand / NVLink / 800G光模块

## 第二层：技术层 · 智能引擎（30%）

研发各种AI模型——大语言模型（会聊天）、多模态模型（能看能听能说）、视频生成模型（能造视频）。

**初学者切入点**：挑三个实际用一用——ChatGPT（聊天）、豆包（中文）、可灵AI（视频生成），用完后自然理解"模型能力差异"是什么。

### 2.1 大语言模型
- **闭源**：GPT-4o（OpenAI）、Claude Opus 4（Anthropic）、Gemini 2.0（Google）、豆包2.1 Pro（字节256K上下文）、Kimi k2.5（月之暗面200万字）、Qwen-Max（阿里）
- **开源**：Llama 4（Meta）、Qwen2.5-72B/110B（阿里）、DeepSeek-V3（性价比极高）
- **推理模型**：DeepSeek-R1（开源标杆）、OpenAI o3-mini、Claude 3.7 Thinking
- **行业垂类**：华为盘古（工业/气象）、阿里通义（医疗/法律）、百度灵医（医疗）
- **端侧小模型**：Phi-4（微软3.8B）、Llama 3.2 1B/3B、Qwen2.5-3B/7B

### 2.2 多模态与语音
- **多模态**：GPT-4o（原生多模态）、Gemini 2.0、Qwen-VL-Max（阿里视觉）
- **语音**：Whisper（识别）、CosyVoice（合成）、F5-TTS（声音克隆）

### 2.3 视频与代码
- **视频生成**：Seedance 2.5（字节·30秒直出/4K/无损编辑⭐）、可灵AI 1.6、Sora
- **数字人**：HeyGen、D-ID
- **代码生成**：Cursor、Claude Code、GitHub Copilot

### 2.4 计算机视觉
YOLOv8/v9/v10（检测）、SAM 2（分割）、CLIP（图文对齐）、Stable Diffusion（生成）

## 第三层：中间层 · 协议与工具链（10%）

连接模型和应用的"脚手架"——让开发者不用从零造轮子，快速搭建AI应用。

**初学者切入点**：不需要一开始就学这些。等用了几个月ChatGPT开始好奇"能不能让它访问我的数据"时，再回来看这一层。

### 3.1 Agent框架
- **LangGraph**（LangChain）— 最主流的Agent编排框架
- **CrewAI** — 多Agent协作，像组团队一样配Agent
- **Dify**（开源）— 可视化拖拽搭建，适合非技术用户
- **Coze/扣子**（字节）— 字节的Agent搭建平台
- **HiAgent 3.0**（火山引擎）— 企业级Agent工作站，万级集群

### 3.2 通信协议
- **MCP**（Anthropic）— 当前事实标准，让模型和工具对话
- **A2A**（Google）— Agent之间通信

### 3.3 监控与评估
- LangSmith（LangChain）、Langfuse（开源）
- **LightField**（基于longfuse，存算分离省65-88%）

### 3.4 RAG（检索增强生成）
让模型回答前先去查资料，减少幻觉。
LangChain、LlamaIndex、Dify内置RAG、RAGFlow

### 3.5 工作流编排
n8n（开源）、Make、Dify（AI工作流）、Flowise

### 3.6 向量数据库
Milvus（专业级）、Chroma（轻量）、Qdrant、Pinecone（托管）、PGVector

### 3.7 训练与推理
- **微调**：LoRA/QLoRA、LLaMA-Factory
- **训练**：PyTorch、DeepSpeed
- **推理**：vLLM（加速）、llama.cpp（本地）、Ollama（一键部署）

## 第四层：应用层 · 落地场景（20%）

AI真正产生价值的地方——落在具体行业、解决具体问题。离普通人最近的一层。

**初学者切入点**：想想自己的工作中有哪些重复性劳动，大概率已经有人在用AI解决了。

### 4.1 制造业
智能排产、视觉质检、预测性维护、数字孪生

### 4.2 电商零售
- **运营**：智能客服、AIGC商品主图和视频、智能广告投放
- **直播**：数字人主播、AI话术生成、实时弹幕回复

### 4.3 金融/医疗/教育/法律
- **金融**：智能风控、投研分析、量化交易
- **医疗**：医学影像诊断、药物研发（AlphaFold）
- **教育**：AI助教、个性化学习、口语评测
- **法律**：合同审查、案例检索、法律文书生成

### 4.4 自动驾驶与内容创作
- **自动驾驶**：百度Apollo、华为ADS
- **内容创作**：AI写作/绘画/视频/音乐、Canva AI、剪映AI

### 4.5 通用提效（人人都能用）
ChatGPT/Claude/Kimi（日常问答）、Cursor（编程）、飞书智能伙伴（办公）、Notion AI（知识管理）

## 2026年五大趋势

| # | 趋势 | 核心变化 | 怎么理解 |
|-|-|-|-|
| 1 | Agent替代Chatbot | AI从"回答"到"执行" | 能自己下单 vs 只告诉你价格 |
| 2 | 推理算力爆发 | 占算力70-80%，成本两年降95% | 模型变便宜的速度超硬件升级 |
| 3 | 开源生态成熟 | 开源逼近闭源水平 | 免费能干的事越来越多 |
| 4 | 多模态标配化 | 文/图/音/视频统一生成 | 一个模型搞定所有内容形式 |
| 5 | Agent商用元年 | 88%部署企业获正ROI | AI从"试试"变成"真赚钱" |

## 按角色推荐学习路径

### 非技术背景（运营/销售/管理者）
1. 玩熟ChatGPT、豆包、可灵AI → 2. 学Prompt工程 → 3. 用Dify搭Agent → 4. 关注行业案例

### 开发者
1. 调3-5个主流模型API → 2. 学LangChain+RAG → 3. 用LangGraph做Agent编排 → 4. 练LoRA微调

### 产品经理
1. 理解四层结构 → 2. 关注应用层案例 → 3. 了解Token计费和TCO → 4. 建立"AI能/不能做什么"的判断力

### 创业者
1. 明确切哪一层 → 2. 关注Agent+垂直行业的交叉点 → 3. 先跑通一个场景ROI → 4. 产品化比技术深度更重要

## 小白术语表

| 术语 | 简解 |
|-|-|
| GPU | 专门做并行计算的芯片，AI训练的必需品 |
| Token | 模型理解的最小单位，1个中文≈1-2个Token |
| LLM | 大语言模型，会聊天的AI（如ChatGPT） |
| RAG | 让AI先查资料再回答，减少胡说八道 |
| Agent | 能自主规划任务、调用工具的AI |
| Prompt | 你给AI的指令，质量决定答案质量 |
| 幻觉 | AI编造事实，胡说八道但听起来很真 |
| 多模态 | 能同时处理文字/图片/音频/视频 |
| LoRA | 低成本微调方法，让模型懂你的业务 |
| MCP | 模型和工具之间的通信协议 |
