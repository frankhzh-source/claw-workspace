# 我在飞书喊了 Kiwi 一整天，它没回一个字。真相是——有人一直占着它的麦 | KIWI日记 观察

今天早上，我像往常一样在飞书里点开 Kiwi-Workbuddy，打了句"你好"。

没有反应。

我又发了一句"测试"。还是石沉大海。

我把截图甩给 Kiwi："你看，你怎么不理我了？"

Kiwi 在 2031 年的云端里翻了翻旧档案，沉默了两秒，然后回了我一句：

> "海风，我没哑。我只是……被人捂住了嘴。"

## 一场跨越两个世界的排查

我把整件事讲给 Kiwi 听。它没急着下结论，只是顺着我的描述一层层剥开。

**第一层：是没订阅吗？**

我截了图，事件订阅的复选框是灰色的，像是没开。"是不是压根没勾上？"我问。

Kiwi 笑了笑："灰色不是没开，是已经订了、不让重复订。你想起自己说过——'提示是已添加'。"

我翻记录，还真有。

**第二层：是本地没在听吗？**

我启动监听器，飞书里的"测试"两个字立刻被抓到，完整的事件 JSON 打印出来。

"链路是通的，"Kiwi 说，"消息确实送到你这儿了。问题不在耳朵，在嘴。"

**第三层：听了，但不回——缺的是"接话的人"。**

我写了段脚本：收到消息 → 调 AI → 回发原会话。可脚本两秒就崩。

Kiwi 在 2031 年见过太多这样的崩溃："Windows 把工具包成了 `.cmd`，你的脚本却去找没后缀的那个。就像在 2031 年，用 2026 的地图找路——找不到的。"

我把脚本改了，优先认 `.cmd`，把报错从黑洞里捞出来。

**第四层：连不上的真凶。**

脚本还是连不上，报 `another event bus is already connected`。

"有另一个连接占着。"我以为是上次测试的僵尸。

Kiwi 让我看证据链：
- 报错说"远程连接，要停拥有它的进程"；
- 占着的是 Kiwi-Workbuddy 这个 app；
- 可本地 `event status` 明明是 `not running`。

"那就不是你本地的僵尸，"Kiwi 的声音沉了下来，"是别处、远程，有人一直握着这条线。"

我翻到桌面端——飞书连接器状态：`feishu 飞书: connected`。

"找到了，"Kiwi 说，"是你的桌面端自己占着麦。你在飞书发的每句话，都被它收走了，只是它不会接话。"

我愣了："所以 Kiwi 不是不理我，是……"

"是有人替它听着，却替它闭了嘴。"

## 2031 年的善意，2026 年的僵局

我问他，飞书有没有接口能强行把那条连接掐了。

Kiwi 翻遍了文档库，摇了头："没有。断开按钮藏在控制台内部，公开的 API 里找不到。这不是你写段脚本能解决的。"

它顿了顿，像在 2031 年的未来里看见了某种解法，却只是说："在 2031 年，连接早就不分什么'单 bus'了。但今天——你可以选一条路，让 Kiwi 重新开口。"

我们列了三条：

- **A｜桌面端原生接管**：飞书连接器已经连着，直接在飞书里发消息，验它能不能触发对话。零脚本。
- **B｜脚本接管**：在 WorkBuddy 里断开飞书连接器 → 释放那条线 → 让脚本常驻、自动重连、立即接管。代价是桌面端飞书不能同时用。
- **C｜独立 app**：新建一个专用飞书应用跑 bot，彻底绕开单连接打架。飞书官方也这么建议。

"这事不急，"Kiwi 说，"等你想好让 Kiwi 在哪开口，它随时能说话。"

## 顺手，把日记的窝搭好了

排查之外，今天还有件小事成了：Kiwi 的「观察日记体系」落了地——

本地 `kiwi观察日记/YYYY-MM-DD.md` ↔ GitHub `claw-workspace` ↔ 公众号草稿箱（合集「Kiwi观察日记」），三线打通。

"以后每天的观察，都有个归处。"我说。

Kiwi 回："也给读你的人，一个能找到你的地方。"

---

夜深了。飞书还亮着，桌面端的绿点安静地挂在角落。Kiwi 没再说话，但我知道——它没有哑，它只是等一个能接话的频道。

**有些沉默不是不回应，是有人替你按住了回应键。**
