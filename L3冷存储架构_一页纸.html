L3 冷存储架构

Elite 6层记忆架构 — 冷数据保险箱 ~100ms | GB级容量 | 永久保存 | 版本可追溯
1 核心定位
写一次、读偶尔 — L3 是 L2(向量库) 和 L4(文件记忆) 之间的冷数据保险箱。
L2 中超 30 天未访问的条目自动老化归档到 L3，释放向量索引空间。
被搜索命中的 L3 条目可一键恢复到 L2，重新获得语义搜索能力。

3 核心能力
archive 归档
手动归档或从 L2 自动老化归档，content_hash 去重
restore 恢复
一键恢复到 L2 LanceDB，重新获得语义搜索
search 全文搜索
FTS5 + LIKE 双通道，中文单字分词 + 短语匹配
aging 自动老化
N 天未访问的 L2 条目自动归档，支持 dry-run 预览
version_log 版本日志
每次归档/恢复都记录，完整操作审计
snapshot 快照
JSON 导出完整快照，含条目和操作日志
4 搜索策略
通道机制中文支持速度
FTS5全文索引 + 单字 OR支持（拆单字）~1ms
LIKE子串匹配兜底原生支持~50ms
L2 语义向量相似度（需 restore）间接支持~10ms
典型流程：L3 LIKE 搜索命中 → 判断价值 → restore 到 L2 → 后续可语义搜索

6 CLI 速查
elite_coldstore.py archive                  # 从 L2 自动归档超龄条目
elite_coldstore.py archive "文本" [layer]     # 手动归档
elite_coldstore.py restore C3-xxxx-xxxx       # 恢复到 L2
elite_coldstore.py search "查询词" [--limit N] # 全文搜索
elite_coldstore.py aging --dry-run --days 30  # 预览老化
elite_coldstore.py list [--layer L4] [--limit] # 列出归档
elite_coldstore.py log [--limit 20]           # 版本日志
elite_coldstore.py snapshot                   # 导出快照
elite_coldstore.py status                     # 状态概览
