LoRA 训练 + ControlNet 控制 — 详解

FDE学习手册 2A进阶B 展开 | 面向电商AI落地实战

一、LoRA 是什么？为什么你需要它？

一句话说清楚

LoRA = Low-Rank Adaptation（低秩适配）。在不改动原模型的前提下，在"旁路"添加一小块可训练的矩阵，让模型学会新的风格/人物/物体/场景。

打个比方：SD模型是一本巨大的百科全书，LoRA就像在书页边缘贴的便利贴。书本身不动，便利贴告诉你"遇到这类图时，往这个方向偏一点"。

# LoRA 的核心公式 原模型输出 = W₀ · x (W₀ 不动) LoRA 修正 = B · A · x (B和A是低秩小矩阵，只有它们被训练) 最终输出 = W₀ · x + B · A · x # 参数量对比 SDXL 模型: ~2.6B 参数（全量微调需要~24GB显存） LoRA 模型: ~10-50MB（一张小文件，训练只需~8-12GB显存）

什么时候用 LoRA，什么时候用全量微调？

场景全量微调LoRA
学一个新风格/画风可以，但浪费最适合
固定品牌视觉风格可以，但浪费最适合
学一个特定人物/产品可以，但浪费最适合
改变模型的基础知识需要做不到
让模型学会全新的概念需要较难（可尝试但效果不稳定）

电商场景 90% 的需求都是 LoRA 能解决的——统一品牌风格、固定产品外观、控制特定场景氛围。

二、训练数据准备（最重要的一步）

数据质量规则

  • 数量：20-30 张高质量图片即可，50 张以上基本到天花板。10 张好图 > 100 张糊图
  • 分辨率：SD 1.5 用 512x512，SDXL 用 1024x1024。所有图片必须统一分辨率，不能混
  • 格式：PNG 或 JPG 都行，推荐 PNG（无损）
  • 多样性：不同角度、不同光照、不同背景，但不能跨类——练"产品风格"就全是产品图，练"人物"就全是人物
  • 一致性：主体占画面 50-80%，不要有裁剪/遮挡/文字/水印

电商训练集准备的三条铁律

  1. 产品图最好是白底或纯色底：让 LoRA 专注学产品本身，别被复杂背景分散注意力
  2. 风格参考图要找最好的那几张：不要拿"凑合"的图来训练，LoRA 会学到"凑合感"
  3. 标注（Captioning）不能省：每张图配上描述文字，这是 LoRA 理解"这张图是什么"的关键

标注（Captioning）怎么做？

每张训练图片需要一个 .txt 文件，里面写描述文字。这是 LoRA 训练中最容易被跳过但最关键的一步。

方式做法适用场景
自动标注用 BLIP/WD14 Tagger 等工具自动生成描述数量多、风格图、通用场景
手动标注人工写下每个图的关键要素产品图（精确度要求高）、人物训练
半自动自动生成 + 人工修正推荐的通用方案
# 标注示例：同一件白色陶瓷杯的不同角度 # white_ceramic_mug_01.txt A white ceramic coffee mug with minimalist design, front view, product photography, studio lighting, white background # white_ceramic_mug_02.txt A white ceramic coffee mug, side view showing the handle, product photography, clean background # 关键原则 # 1. 描述"图里有什么"，不要描述"这是什么东西"（那是触发词做的事） # 2. 品牌/产品名不放描述里，单独做触发词 # 3. 每张图独有特征（角度/光照/细节）写清楚，共性特征（产品材质/风格）可以省略

三、核心训练参数详解

参数推荐值含义调高/调低的后果
Epoch（轮数） 10-20 训练集完整过多少遍 太少：没学到，出图和训练集完全不像
太多：过拟合，除了训练集的内容其他都不会画了
Learning Rate（学习率） 1e-4（0.0001） 每次更新参数时迈多大步 太低：学太慢，20个epoch还像没训
太高：一步太大直接跑偏，画面崩溃/颜色爆炸
Batch Size（批次大小） 1-4 一次喂几张图给模型 太小的Batch：训练不稳定，曲线抖动大
太大的Batch：显存不够（OOM）
Network Rank（秩） 8-32 LoRA 矩阵的"容量"，越大能学的东西越多 Rank 4-8：轻量风格、氛围微调
Rank 16-32：人物/产品固定、复杂风格
Rank 64+：很少需要，文件大但提升有限
Network Alpha = Rank（或 Rank/2） LoRA 生效强度的缩放因子 Alpha 越小 → LoRA 效果越弱（出图更接近原模型）
Alpha = Rank 是标准配置
Optimizer（优化器） AdamW8bit 用什么算法更新参数 AdamW8bit 省显存且效果稳定，不要随意换
LR Scheduler cosine 学习率的变化曲线 cosine = 先大后小，稳步收敛。constant = 全程不变，容易震荡

新手推荐配置（拿来就能用）

# SD 1.5 模型 LoRA 训练推荐配置 Resolution: 512x512 Epoch: 10 Learning Rate: 1e-4 Batch Size: 2 Network Rank: 16 Network Alpha: 16 Optimizer: AdamW8bit LR Scheduler: cosine # SDXL 模型 LoRA 训练推荐配置 Resolution: 1024x1024 Epoch: 10 Learning Rate: 1e-4 Batch Size: 1 Network Rank: 16 Network Alpha: 16 Optimizer: AdamW8bit LR Scheduler: cosine # SDXL 对显存要求更高，Batch Size 老老实实设1

四、训练工具选型

工具特点适合谁上手难度推荐度
kohya_ss 功能最全、教程最多、社区最活跃。支持 SD1.5/SDXL/FLUX 的 LoRA 训练 大多数人（入门到进阶） 中等 首选

kohya_ss 训练流程（最简版 8 步）

  1. 准备图片：20-30 张统一分辨率的 PNG，放到一个文件夹
  2. 标注图片：每张图片配一个同名 .txt 描述文件（用 BLIP/WD14 自动 + 人工复核）
  3. 启动 kohya_ss：双击 gui.bat 或运行 `python kohya_gui.py`
  4. 选择 Source Model 标签页：选你的基础模型（SD1.5 的 .safetensors 文件），设置输出目录
  5. 选择 Folders 标签页：指定图片文件夹路径，设置 Resolution 为 512 或 1024
  6. 选择 Parameters 标签页：按上面的新手配置填入参数
  7. 点 Start Training：等着。10-20 epoch 在 RTX 3060 12G 上大概 10-30 分钟
  8. 测试 LoRA：把生成的 .safetensors 文件放到 SD 的 models/Lora/ 目录，Prompt 里加上 `<lora:你的LoRA名:0.8>` 测试效果

五、ControlNet — 精准控制构图的武器

为什么需要 ControlNet？

文生图最大的痛点是不可控——你写了一堆 Prompt，出来的构图、姿势、细节全凭运气。ControlNet 给你一个"方向盘"：上传一张参考图（可以是线稿、骨架、深度图），告诉 SD"按这个结构画，内容你自由发挥"。

六种核心模式对比

模式输入什么控制什么电商最佳场景推荐强度
Canny 边缘检测图（黑白线条） 物体的外形轮廓 产品形状不变，换背景/材质/风格 0.7-0.9
Lineart 线稿图（比Canny更干净） 物体的线条结构 服装/鞋包等轮廓清晰的品类 0.7-0.9
Depth 深度图（灰度，越白越近） 物体的空间位置和透视 场景图（产品在桌子/沙发上） 0.5-0.8
OpenPose 人体骨架图 人物的姿势 模特展示图（控制站姿/手势） 0.8-1.0
IP-Adapter 参考图（人脸/风格） 图像整体风格、面部特征 保持模特面部一致、品牌风格一致 0.5-0.8
SoftEdge 柔和边缘图 松散的轮廓控制 不想太死板、需要一点自由度时 0.4-0.6

电商场景最佳实践：产品图流水线

# 商品图标准化流水线（ComfyUI 节点流程） 步骤1: 输入产品白底图 步骤2: ControlNet Canny → 提取产品轮廓 步骤3: 加载品牌 LoRA（之前训练好的） 步骤4: 加载场景 LoRA（咖啡厅/卧室/户外...） 步骤5: Prompt: "A [产品名] on a wooden table, morning sunlight, cozy cafe..." 步骤6: SD 生成 → 产品形状不变 + 品牌风格一致 + 场景替换成功 # 模特换装流水线 步骤1: 输入参考姿势图（从目标模特/网图提取） 步骤2: ControlNet OpenPose → 提取骨架 步骤3: ControlNet IP-Adapter → 固定模特面部 步骤4: 加载服装 LoRA（目标服装训练的） 步骤5: 加载场景/风格 LoRA 步骤6: Prompt: "A fashion model wearing [服装], walking on a city street..." 步骤7: SD 生成 → 姿势对 + 脸没变 + 衣服对 + 场景对

六、常见踩坑与解决

问题现象解决方案
过拟合 LoRA 强度开 0.5 以上就崩，只能出训练集里的图，不会泛化 ①减少 Epoch（10->7）②提高学习率让训练更"粗糙"③增加训练集多样性④降低 Network Rank
欠拟合 LoRA 开到 1.0 了还是不像，出图跟没加载 LoRA 差不多 ①增加 Epoch②降低学习率③检查标注是否太笼统④数据集质量差（图片模糊/大小不一）
色彩偏色 所有出图都偏某个颜色（如发黄/发蓝） 训练集照片的色温不一致导致。统一调色后再训练；或降低 Network Alpha 减弱 LoRA 影响
LoRA 和基础模型不兼容 用 SDXL 练的 LoRA 放到 SD1.5 模型上用，出图全乱 LoRA 必须和训练时用的基础模型版本一致。SD1.5→SD1.5，SDXL→SDXL，FLUX→FLUX
CUDA Out of Memory 训练到一半显存爆了 ①降 Batch Size 到 1②降分辨率③用 AdamW8bit 代替 AdamW④勾选 Gradient Checkpointing⑤换更大显存的显卡
标注错误 Prompt 里写了某个词，LoRA 反而把那个东西画错了 标注时把"你想要 LoRA 学会的东西"从描述里去掉，只保留背景/环境/角度等。比如练杯子，描述里就不要出现"white cup"，只写"product photography, studio lighting"

七、LoRA + ControlNet 组合策略

四种经典组合方案

方案组合解决的问题电商案例
风格固定 品牌 LoRA（强，0.8-1.0） 每次出图的色调/质感/氛围统一 全店商品图风格一致，像同一个摄影师拍的
产品固定+场景换 产品 LoRA（强） + ControlNet Canny + 场景 Prompt 产品不变，换不同背景/场景 同一个杯子，今天出咖啡厅场景图，明天出户外野餐场景图
模特固定+服装换 IP-Adapter（面部） + OpenPose + 服装 LoRA 同一个模特穿不同衣服，姿势可控 品牌签约模特的AI替身，不用真人反复拍
全要素控制 品牌 LoRA + 产品 LoRA + ControlNet + IP-Adapter 风格、产品、构图、面部全部可控 高端品牌的完整产品详情页素材生产

⚠ 全要素控制的 LoRA 不能堆太多（3个以上容易互相打架），且推理速度会显著下降。建议一个场景不超过 2 个 LoRA + 1 个 ControlNet。

学习路径建议

第一步（1天）：用 kohya_ss 跑通第一个 LoRA，选最简单的——比如练自己桌面的杯子，10 张图，默认参数，先感受一下训练是什么
第二步（2天）：用品牌的产品图训一个品牌 LoRA，调参到出图风格稳定统一
第三步（2天）：把品牌 LoRA + ControlNet Canny 组合，实现产品固定+场景替换
第四步（3天）：扩展到 ComfyUI 工作流，实现 CSV 批量调用，产出 50+ 张统一风格图

两周内把这条流水线跑通，你就有了电商 AI 生图的核心交付能力。
