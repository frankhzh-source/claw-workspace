1 记忆分层（三层模型）
层级名称保留周期存储介质典型实现延迟目标
L1工作记忆 Working Memory会话内（分钟~小时）进程内存 / RedisPython dict / Context Window<1ms
L2短期记忆 Short-term Memory数小时~数天DB + 向量索引SQLite + FAISS / Chroma~10ms
L3长期记忆 Long-term Memory永久向量DB + 知识图谱Milvus/Qdrant + Neo4j/Nebula~100ms
分层原则：时间局部性（最近访问保留在更快的层）→ 空间局部性（相关概念聚集存储）→ 容量递增 L1<L2<L3 → 自动沉降 L1→L2→L3 渐进式归档
2 核心模块 & 流程
📝 2.1 写入模块（Write Pipeline）
原始输入 → 清洗去重 → Embedding → 路由分层 → 更新/追加
① 清洗
去除 PII、噪声、重复内容
② 向量化
BGE / OpenAI text-embedding-3 / Cohere v4
③ 路由决策
内容类型+时效性 → L1/L2/L3
④ 增量更新
append / merge / overwrite 三模式
🔍 2.2 检索模块（Retrieve Pipeline）
用户Query → Query改写 → 多路召回 → 重排序 → 上下文组装
召回策略技术方案说明
语义召回Vector Search余弦相似度 > 阈值
关键词召回BM25 / TF-IDF精确匹配
知识图谱扩展实体关系遍历跳数限制 2-3 hop
时序过滤时间衰减加权优先 recent memories
🗜️ 2.3 压缩模块（Compression）
触发条件压缩方式目标压缩比
L2 容量超阈值LLM 摘要（保留关键事实+决策理由）5:1 ~ 10:1
相似记忆堆积合并去重 + 抽象升级3:1 ~ 5:1
长对话归档结构化提取（实体/动作/结果）10:1 ~ 20:1
⚠️ 压缩必须保留可追溯性：压缩后的记忆需记录来源 ID 和压缩时间戳
♻️ 2.4 生命周期管理（Lifecycle）
老化策略
访问频率 × 时间衰减 = 生存分数，低于阈值则归档/删除
合并策略
相似度 > 0.9 的记忆条目自动合并
审计日志
所有写操作记录 who/when/what，支持回滚
遗忘机制
用户显式「忘记」某段记忆（GDPR 合规）

4 生产级避坑要点
🔴 必须避免
  • 不要把全部上下文塞进 Prompt — 成本爆炸 + 质量下降（lost-in-the-middle）
  • 不要忽略 Embedding 域偏移 — 通用模型专业领域差，需 domain fine-tune
  • 不要同步阻塞主流程做写入 — 必须异步队列解耦
  • 不做检索就全量返回 — 即使 1 万条也不行，必须 top-K
🟡 推荐做法
  • 采用 Hybrid Search：向量+关键词混合召回，覆盖率 +30%
  • 设置 Recall Guardrail：低于阈值宁可少召也不要噪音
  • 实现 Observability Dashboard：命中率/延迟P99/压缩率可视化
  • 设计 Fallback 降级：L3 不可用时降级到 L2 全量扫描
🟢 加分项
  • 记忆反思机制（Reflection）：定期回顾发现矛盾并修正
  • 用户反馈闭环：「有用/无用」进入强化学习信号
  • 跨会话一致性校验：冲突时触发告警+人工审核
  • 多模态记忆支持：图片/代码块/表格的结构化存储检索
