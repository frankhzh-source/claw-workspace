Phase 1：基础能力

不依赖任何场景，所有方向通用

1.1 分步推理 Prompt 工程

核心知识

  • Prompt = 指令 + 上下文 + 输出格式。三者缺一不可，缺指令模型乱猜，缺上下文模型瞎编，缺格式输出不可控
  • Few-shot（示例引导）：给2-5个示例告诉模型"长什么样"。示例比指令更管用——模型更擅长模仿而非遵循规则
  • Chain-of-Thought（分步推理）：让模型先分析再回答，不要一步到位。加一句"请逐步分析，列出思考过程，最后给出结论"可以显著提升准确率
  • 角色设定："你是XX领域的专家，有10年经验"不是废话。角色设定会影响模型的用词风格和知识范围
  • 约束条件：字数限制、禁用词、必须包含的要素——越具体越好。"写200-300字，包含至少3个卖点，不用'超值''限时'等过度营销词"
  • 输出格式控制：要求JSON/Markdown表格/列表格式。如果模型不听话，加一句"只输出JSON，不要任何解释"

实战示例：同一产品写5个平台文案

# 系统指令（System Prompt） 你是一个电商文案专家。你的任务是根据商品信息，为不同平台撰写适配的文案。 # 商品信息 产品：便携式挂烫机 核心卖点：①30秒快速预热 ②迷你便携，出差可用 ③干湿两用 ④价格149元 目标人群：25-35岁职场女性 # 任务：为以下平台各写一段文案，符合各平台风格 格式要求：用Markdown表格输出，列名：平台 | 文案 | 字数 各平台要求： - 淘宝标题：30字以内，堆核心关键词，突出搜索权重 - 抖音口播文案：100-150字，口语化，有钩子+痛点+解决方案+CTA - 小红书：轻松种草风，50-80字，带场景描述 - 拼多多：30字标题+一行描述，走"便宜实用"路线 - 微信朋友圈：60-80字，带人设感，像朋友推荐

进阶技巧：Chain-of-Thought 应用于电商场景

# 不用 CoT 的问法（差） "这个产品怎么定价？" # 用 CoT 的问法（好） "请分三步分析这个产品的定价策略： 第一步：分析竞品价格区间和市场定位 第二步：计算我们产品的成本结构和利润空间 第三步：给出3个定价方案，并说明每种方案的适用场景和风险"

1.2 AI 图片基础

核心知识

  • 两大流派：MidJourney（傻瓜式，月费制，审美天花板） vs Stable Diffusion（开源本地部署，完全可控，需要显卡）
  • Prompt 公式（五要素法）：主体 + 场景/环境 + 风格/材质 + 光照 + 画质/比例。缺哪一项就不可控哪一项
  • 负向提示词（Negative Prompt）：告诉模型"不要什么"。SD必用，MJ不需要。常见负向词：low quality, blurry, extra fingers, watermark, text
  • CFG Scale（引导强度，SD专用）：数值越高越遵从Prompt，太高会过饱和/失真。商品图建议5-8，艺术创作7-12
  • 采样器（Sampler，SD专用）：Euler A 快速但随机性强，DPM++ 2M Karras 细节多但慢，选一个稳定用
  • Steps（步数，SD专用）：20-30步足够，超过35步收益递减，纯烧时间
  • 图像尺寸：水平构图的图用横版，模特/单品用竖版。常见电商尺寸：800x800（主图）、750x1000（详情页首图）、1080x1920（抖音竖版）
  • 商品抠图换背景：用 Remove.bg（在线）/ Rembg（Python库）/ PS AI / Segment Anything 抠出商品，再放到AI生成的场景图中

实战 Prompt 模板

# 白底商品图（适合主图） 主体：[产品名称，如：白色陶瓷咖啡杯] Prompt: A [产品], product photography, white background, studio lighting, 8k, high detail, commercial photography, centered, isolated Negative: shadow on product, reflection, dark background, watermark # 场景图（适合详情页/广告） Prompt: A [产品] placed on a wooden table near a sunny window, morning light, cozy home atmosphere, lifestyle photography, 4k, shallow depth of field, minimalist interior style Negative: blurry, dark, cluttered, text, watermark # 模特展示图 Prompt: A young woman wearing [服装名称], full body shot, walking on a city street, natural sunlight, fashion photography style, clean background, 8k Negative: deformed face, extra limbs, bad anatomy, watermark

  • Stable Diffusion WebUI (A1111) 安装教程 — B站搜"秋叶SD整合包" 中文首选

1.3 数据工具

核心知识

  • 生意参谋（淘宝天猫）：看"流量-商品来源"知道搜索/推荐/付费占比；看"品类-商品排行"知道哪个款在起量/掉量；看"竞争-竞品分析"对标竞品流量结构
  • 抖店罗盘（抖音电商）：看"直播-直播大屏"实时监控GPM/转化率/停留时长；看"商品-商品卡"分析搜索曝光→点击→转化的漏斗；看"人群-人群画像"了解谁在买
  • 关键指标速查：
    CTR（点击率）= 点击数 / 曝光数主图/标题好不好的第一指标
    CVR（转化率）= 订单数 / 点击数详情页/价格/信任感好不好的指标
    GPM（千次观看成交额）= GMV / 观看次数 × 1000抖音直播核心指标
    客单价= GMV / 订单数人群质量 + 连带率
    ROI（投产比）= GMV / 广告花费花钱效率
  • Excel + AI 分析三板斧：①把数据导入Excel → ②让AI写分析脚本（Python/VBA）→ ③让AI解读结果并给出建议

实操：用AI辅助诊断店铺问题

给AI发以下内容，让它诊断：

"我有以下店铺数据，请帮我做诊断分析： 近7天数据： - 曝光：50,000（上周45,000，涨11%） - 点击：1,500（上周1,350，涨11%） - 加购：150（上周180，降17%）← 关键异常 - 下单：45（上周60，降25%）← 关键异常 - 广告花费：3,000元 - 行业平均CTR：3.2%，CVR：3.5% 请分析： 1. 流量来了但转化掉了，可能的原因有哪些？ 2. 加购率下降说明什么问题？ 3. 给出3条优先级最高的行动建议"

1.4 工具选型方法论

评估框架：五维度打分法

维度评分标准（1-5）权重建议
效果质量1=不可用，3=可用但需人工修，5=超过人工水平35%
使用成本1=每月上万，3=几百，5=免费/极低25%
上手难度1=需要专业技术背景，3=看文档能搞定，5=打开即用15%
稳定性/速度1=频繁掉线/慢，3=偶尔卡，5=秒级响应15%
平台适配1=完全不兼容，3=需要额外转换，5=原生支持10%

选型铁律：没有完美工具，只有最适合当前场景的工具。一个工具选好了用一年，选错了浪费两周。

Phase 2：单场景精通

2A 文生图（电商视觉）

入门：选型 + Prompt 模板库

  • 选型决策树：做概念图/效果图→MJ | 做批量商品图/品牌风格控制→SD+ComfyUI | 只是偶尔用用→通义万相/DALL-E | 预算充足但不想折腾→MidJourney
  • Prompt 模板库搭建方法：按"品类+构图+风格+光照"四标签分文件夹，每次用过的Prompt存下来，标注效果好坏。一个月后你就有自己的武器库
  • 商品图常用的构图：正面特写（主图）、45度角（细节展示）、场景融入（详情页）、手持/上身（模特展示）、配件搭配（连带推荐）

进阶A：本地部署 Stable Diffusion

  • 选择 A1111 WebUI 还是 ComfyUI？A1111 界面直观、教程多，适合入门；ComfyUI 节点化、批处理和管线更强大，适合量产 推荐A1111入门→ComfyUI进阶
  • GPU 要求：SD 1.5 最低4GB显存，SDXL 最低8GB，FLUX 最低16GB。建议至少RTX 3060 12G起步 苹果M系列Mac也能跑，慢但不挑
  • 模型文件位置：SD模型(.safetensors)放 models/Stable-diffusion/，LoRA(.safetensors)放 models/Lora/，VAE放 models/VAE/
  • 基础模型推荐：SDXL 写实用 Juggernaut XL / RealVisXL，动漫/风格化用 Animagine XL，电商通用 SD 1.5 的 majicMIX realistic
  • 踩坑清单：①CUDA Out of Memory→降低分辨率或换小模型 ②生成纯黑图→VAE没放对 ③颜色失真→关闭VAE或换VAE

进阶B：LoRA 训练 + ControlNet 控制

  • LoRA 训练核心参数：数据集建议20-30张高质量图片，分辨率512x512或768x768，epoch 10-20，学习率1e-4左右。数据质量 > 数量——10张好图比100张糊图强
  • 训练工具：kohya_ss GUI（主流，图形界面）/ OneTrainer（更现代）/ 秋叶整合包内置 推荐kohya_ss
  • ControlNet 常用模式：Canny（边缘检测，保持物体形状）| Depth（深度图，保持空间透视）| OpenPose（骨架，控制人物姿势）| Lineart（线稿，电商用这个最多）
  • 电商场景最佳实践：产品白底图→Canny提取轮廓→用ControlNet保持形状不变→换背景/场景/模特

实战：批量自动化生产

  • ComfyUI 工作流自动化：读CSV商品列表→切换LoRA→生成多张→自动保存→人工质检。用ComfyUI的API接口 + Python脚本触发
  • 质检标准：①主体完整（无残缺/变形/多余）②色彩准确（与实物一致）③构图可用（留白足够放文字）④风格统一（同批次不跳跃）
  • 人工审核降50%的秘诀：每个SKU生成5张选1张，而不是生成1张修1张。多生成+快筛选 > 精修

  • civitai.com 看 LoRA 效果 — 搜"product""e-commerce""fashion"

2B 文生视频

入门：工具选型 + 模板

  • 主流工具对比：
    工具时长质量价格适合场景
    可灵 (Kling)5-10秒高会员制中文Prompt友好，人物动作自然
    即梦 (Jimeng)3-5秒中高免费额度多字节系，抖音生态最好
    Runway Gen-35-10秒高按秒计费画质最好，风格最多
    Pika3秒中免费/订阅快速出片，适合社交媒体
    Sora (即将开放)60秒最高未定价长视频，叙事型内容
  • 15s/30s/60s 模板结构：15s（3秒钩子+8秒展示+4秒CTA）| 30s（3秒钩子+20秒展示+7秒CTA+品牌）| 60s（5秒钩子+15秒场景痛点+30秒产品解决+10秒CTA）
  • 视频 Prompt 写法：时间轴描述法——"0-3秒：XX动作。3-8秒：镜头推进到XX。8-15秒：XX特写。" 越具体AI越听话

进阶：图生视频 + 混剪

  • 图生视频最佳场景：①静态商品图→旋转/展示动画 ②产品场景图→镜头推拉/光影变化 ③人物+产品图→简单动作（穿/用/拿）
  • 实拍+AI混剪黄金比例：AI视频占30-40%（场景氛围/特效/过渡），实拍占60-70%（产品特写/人物口播/使用演示）。AI太重会让观众觉得"假"
  • 后期工具链：AI生成片段→剪映/CapCut拼接→AI配音（剪映内置/魔音工坊）→AI字幕→调色统一

实战：信息流投流素材 + 平台算法

  • AB测试框架：同一个产品出10条素材，每条只有1个变量不同（钩子文案/画面风格/时长），投500元测CTR，筛出TOP3再放大预算
  • 抖音信息流核心逻辑（嵌入学习）：
    • ECPM = 出价 × 预估CTR × 预估CVR × 1000。你的出价一样，CTR决定生死
    • 完播率权重 > 点赞权重：让观众看完比让观众点赞更重要
    • 3秒完播率（前3秒没划走的比例）：决定冷启动流量池大小，低于30%几乎不可能跑量
    • 15s视频的完播率要求 > 60s：越长的视频，完播率天然越低，平台会按比例折算
  • 素材优化的5个杠杆：钩子（0-3秒）> 画面冲击力 > 产品卖点清晰度 > BGM > 口播语速。前两个定生死，后三个定转化

2C 智能客服

入门：知识库 + 意图分类

  • 知识库搭建步骤：①收集高频问题（翻最近1个月客服聊天记录）→ ②归类（按售前/售后/物流/投诉）→ ③每类写标准答案（不要抄文档，要用客服真实话术）→ ④RAG注入
  • 意图分类体系（电商版）：
    大类子意图典型话术策略
    售前咨询商品信息/尺寸/材质"这个多大？"RAG直接回答
    售前咨询比价/议价"能便宜点吗？"话术引导+优惠券
    售前咨询推荐/选择困难"A和B哪个好？"对比分析+引导决策
    订单查询物流/发货/改地址"到哪了？"API实时查询
    售后问题退换货/质量问题"收到了坏的"安抚+标准化流程
    售后问题投诉/差评"我要投诉"升级转人工
  • FAQ结构化原则：一个问题对应一个确切答案，不要多个问题混一起。问题用客户原话（不是你的话），答案要包含具体参数（尺寸/政策/链接）

进阶A：平台客服API基础

  • 抖音小店开放平台：主要接口 — 获取会话列表/发送消息/查询订单/查询物流。接入方式：创建应用→获取授权→调用API 自研应用模式
  • 淘宝开放平台（TOP）：接口最全但准入最严，需要企业资质和API权限申请。核心接口：taobao.trade.get(查订单)、taobao.logistics.trace.search(查物流)
  • 京麦开放平台：通过京东商家后台→服务市场→自研应用接入，接口风格类似淘宝TOP
  • 关键坑：每个平台的API限频（QPS限制）不同，抖音最松、淘宝最严。大促期间可能临时收紧限频，需要熔断机制

进阶B：对话流程 + 情绪检测 + 人工接管

  • 多轮对话流程图设计：用流程图工具（Mermaid/Draw.io）画出每个意图的对话分支——用户说了X→AI应该问Y或执行Z→用户回复A/B/C→分别处理。不设计就直接上线=百分之百翻车
  • 情绪检测信号：感叹号连续(!!) / 反问句 / 辱骂词 / 重复同一问题3次以上 / "人工""投诉""差评"关键词。检测到任一信号→启动安抚话术+准备转人工
  • 人工接管策略：不要等到用户说"转人工"才转，AI检测到情绪上升时主动提议"需要我帮您转接人工客服吗？"——用户感觉被尊重，转接率反而降低
  • 转人工率公式：= 转人工次数 / 总会话数。业内标杆<10%，初期<15%可接受

实战：质检自动化

  • 质检维度：①准确性（答错/误导）②完整性（漏回答/答非所问）③态度（生硬/冷漠）④合规（虚假承诺/泄露隐私）
  • 自动化方案：用另一个LLM（不同模型，防止"自己审自己"）抽检10%的对话，按上述4维度打分。低于阈值标记人工复核

2D 直播运营

入门：直播脚本结构 + AI生成

  • 5分钟切片法：把2小时直播切成24个5分钟单元，每个单元 = 钩子(30秒) + 产品讲解(2分钟) + 逼单(1分钟) + 互动/福利(1分钟) + 过渡(30秒)
  • 脚本Prompt模板：给AI"产品信息+直播时长+排品顺序+主播风格"，让它按5分钟切片法生成完整脚本。例如："你是一个抖音带货主播'海风'，风格真实直接，不做作。今晚直播2小时，排品：①引流款-9.9元袜子（前30分钟）②利润款-149元挂烫机（30-90分钟）③福利款-59元T恤（90-120分钟）。请按5分钟切片法生成完整脚本"
  • 排品策略黄金公式：引流款（低客单拉人）→ 利润款（中客单做GMV）→ 福利款（限量逼单）→ 引流款（再次拉人）→ 利润款... 循环

进阶：实时数据 + 弹幕 + 话术优化

  • 直播核心数据指标：
    指标含义健康值低了怎么办
    在线人数当前在直播间的人数越高越好检查推流/投放/内容
    停留时长人均观看时长（秒）>90秒钩子不够强，换话术
    互动率评论/点赞/分享人数/在线>3%主动点名互动、发福利
    GPM千次观看成交额>500元逼单力度不够，话术优化
    商品点击率点小黄车的比例>5%讲解不到位，重新展示
  • 实时提词系统搭建：后台监控弹幕高频词→AI自动生成回复→推到主播面前的iPad/提词器→主播照读即可。核心是"快"——从弹幕出现到提词出现不超过5秒
  • 流失节点识别：每5分钟看一次在线人数变化。如果某5分钟在线陡降→回放那个时段的录播→分析是不是话术/产品/画面出了问题

实战：AI切片 + AB话术

  • AI切片流程：录播文件→AI识别高能时刻（在线峰值/互动爆发/成交密集）→截取15-30秒→自动加字幕+BGM→批量发布到账号。这是把2B文生视频能力复用到直播场景
  • AB话术测试：同一个产品，准备A版话术和B版话术（只改1-2个变量），在两场直播中分别使用，对比转化率。累计10场数据后，你有自己的话术方法论

2E 数字人主播

选型搭建：平台对比 + 形象克隆 + 推流

  • 主流平台对比：
    平台优势劣势价格
    硅基智能中文形象最多、定制流程成熟形象偏"AI感"按分钟/年费
    闪剪图文转视频、模板多不支持实时直播互动订阅制
    腾讯智影腾讯生态、合规性最好功能较少按量
    快手数字人平台原生、推流稳定仅快手生态免费/分成
    HeyGen英文效果最好、形象最自然中文口型一般、贵按分钟
  • 形象克隆流程：录制3-5分钟正脸视频（光线均匀、背景纯色、语速正常）→上传到平台→AI训练（1-24小时不等）→生成数字人→测试口型和动作→迭代优化
  • 推流全流程：数字人软件→虚拟摄像头(OBS)→推流地址(抖音/淘宝的RTMP地址)→开始直播。关键：帧率>25fps、分辨率>720p、码率>2000kbps
  • 声音克隆：GPT-SoVITS（开源免费，中文效果最好）| CosyVoice（阿里开源，速度快）| ElevenLabs（付费，英文效果最好但中文口型不自然）

优化运营：数据 + 协作

  • 数字人特有指标：①场观流失率（多少人在前30秒走了）>真人主播，因为AI感 ②互动率 << 真人，必须用弹幕互动弥补 ③挂机稳定性——数字人核心优势，7x24不累
  • 真人+数字人协作排班：黄金时段（晚7-11点）用真人 → 白天/凌晨用数字人 → 数字人跑不动的品换真人 → 真人AB测试后的话术喂给数字人
  • 合规要点（嵌入式学习）：抖音要求数字人直播间必须标注"虚拟主播"，淘宝要求标注"AI生成"，不标注会被限流/封禁。各平台政策不同且经常更新，需要每月确认 高风险管理

矩阵扩展：多账号 + ROI

  • 一拖多模式：一个数字人形象 + 一套话术 + 多个账号 = 同时覆盖不同品类/时段。每个账号做微调（封面/标题/开场白），避免平台判定为重复内容
  • ROI核算公式：数字人月成本（平台费+电费+设备折旧） v.s. 带来的GMV增量 - 真人主播在该时段可能产生的GMV。数字人覆盖的是真人做不到的时段，增量GMV才是真ROI

2F 电商运营 AI 化

入门：AI选品 + 标题 + 详情页

  • AI选品四步法：①数据源对接（生意参谋市场洞察/抖音罗盘行业数据/拼多多热搜词）→②AI分析趋势（哪些品类搜索量上升/哪些在掉）→③交叉验证（数据趋势 × 供应链可行性 × 利润率）→④AI生成选品报告
  • 消费者洞察嵌入：选品本质是选"人"不是选"品"。分析竞品评论区高频词 = 消费者需求；分析搜索词 = 消费者痛点；分析竞品差评 = 市场空白 这是5B消费者洞察的实战化
  • 标题优化公式：核心词 + 属性词 + 场景词 + 人群词 + 品牌词。例如"【海风优选】便携挂烫机 迷你出差手持熨斗 小型家用宿舍蒸汽电熨斗"。AI生成10版→人工筛选3版→AB测试1周→锁定最优版
  • 详情页AI化：AI提取产品卖点→按"痛点-解决方案-产品优势-信任背书-行动号召"结构化→配图标注位置→批量生成→人工校对

进阶：定价 + 库存 + 竞品监控

  • 动态定价模型：基于竞品价格 + 自身成本 + 转化率弹性，AI给出建议价。不是"AI定价"，是"AI建议→人决策"
  • 库存预测：用过去90天的销量数据 + 大促日历 + 季节性因子，AI预测未来30天各SKU的销量，指导备货。关键：AI不知道供应链突然断了，必须人工复核
  • 竞品自动化监控：Python脚本定时爬竞品价格/主图/标题/销量变化→AI分析变化趋势→发现竞品降价/换主图/上新→自动推送到飞书/钉钉

实战：全链路闭环

  • 全链路 = 选品→视觉→上架→推广→客服→复购。每个环节AI化后串联起来，实现"一个新品从选品到复购，AI参与每个节点的建议或执行"
  • 效率提升衡量：选品时间（人天） / 上架时间 / 客服响应时间 / 复购率。上线前后各测一周，对比差值
  • 踩坑清单：①AI全取代→效果差→要人机协作 ②只看效率不看效果→为了快牺牲转化 ③数据没打通→各环节数据孤岛，AI只能局部优化

3.2 ROI 核算

ROI 计算模板

  • 公式：ROI = (AI带来的增量价值 - AI成本) / AI成本
  • 增量价值 ≠ 省了多少人工。AI 24小时在线的增量、AI批量的增量、AI从0到1做新场景的增量，都要算进去
  • 不可量化的价值也要列出来：响应速度提升、品牌调性统一、团队AI能力积累、客户体验优化
# 2A 文生图月ROI 月增量价值 = (AI出图数 - 原人工出图数) × 单图价值 + 省掉的拍摄费用 AI月成本 = API调用费 + 本地GPU电费 + 人均操作时间 月ROI = (月增量价值 - AI月成本) / AI月成本 # 示例 月增量价值 = (500图-200图)×20元 + 省拍摄费5000元 = 11000元 AI月成本 = 200元API + 300元电费 + 50小时×50元 = 3000元 月ROI = (11000-3000)/3000 = 2.67（投资1元回报2.67元）

3.3 流程标准化（SOP）

SOP 标准模板

  • SOP必备要素：①前置条件（开始前需要什么）②操作步骤（每一步做什么+截图/录屏）③常见问题（FAQ）④交付标准（做到什么程度算完成）⑤检查清单
  • 2A文生图SOP示例结构：环境准备（SD已启动+模型已加载）→接收需求（SKU编号+品类+风格要求）→生成（Prompt模板+生成5张）→质检（按4项标准打分）→输出（命名+归档+上传）
  • SOP不是写一次就完的。每次操作发现有更好的方法，立即更新SOP。SOP就像代码，需要维护和迭代
