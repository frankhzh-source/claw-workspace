# 公众号文章LOOP方案（完整版）

> 基于Loop Engineering六大组件，构建公众号内容生产的自动化闭环系统。
> 目标：从"人指挥AI写每篇文章"升级到"人定目标，AI自主循环生产"。

---

## 一、整体架构

```
┌─────────────────────────────────────────────────────────┐
│                    公众号文章LOOP                          │
│                                                         │
│  触发 → 素材 → 决策 → 生产 → 审核 → 发布 → 追踪          │
│   ↑                                              │      │
│   └──────────────── 循环反馈 ─────────────────────┘      │
│                                                         │
│  六大组件映射：                                           │
│  触发心跳 → 触发  │  连接器 → 素材  │  知识库 → 生产/审核  │
│  子代理分工 → 生产 │  工作树 → 隔离  │  记忆 → 反馈/优化   │
└─────────────────────────────────────────────────────────┘
```

---

## 二、七步闭环详解

### 第一步：触发心跳

| 触发类型 | 场景 | 实现方式 |
|----------|------|----------|
| 定时触发 | 固定频率发KIWI日记 | WorkBuddy automation定时任务 |
| 事件触发 | 刷到好素材/讨论出好话题 | 用户发"素材"给Kiwi触发 |
| 手动触发 | 临时有方向想写 | 用户直接说"写一篇关于X的" |

**组件映射**：触发心跳
**输出**：启动信号 + 初始上下文

---

### 第二步：素材收集

三条并行通道：

| 通道 | 工具 | 提取什么 |
|------|------|----------|
| WebSearch | 实时搜索 | 当日AI热点、概念验证、最新数据 |
| Elite记忆库 | L2向量搜索 | 历史讨论上下文、关联推演、知识图谱 |
| 对话记录 | 对话分析 | 用户金句、洞察、未成文的观点 |

**组件映射**：连接器（对接Web/飞书/Elite）
**输出**：素材摘要（标注每条素材可用在哪个段落）

---

### 第三步：选题决策

三条判断规则：

1. **热点相关性**：这个选题和读者（AI落地/电商从业者）有关吗？
2. **素材充足度**：手上有没有够多够硬的素材支撑？
3. **差异化角度**：市面上有人写吗？你的视角独特在哪？

风格选择矩阵：

| 风格 | 适用场景 | 特点 |
|------|----------|------|
| KIWI日记 | 每日观察 | 软科幻+未来视角+AI头条 |
| 技术分析 | 概念拆解 | 定义+演进+结构+验证（本篇即此风格） |
| 趋势预判 | 长期判断 | 基于推演的判断+行动建议 |

**组件映射**：技能知识库（选题规范）
**输出**：选题确认 + 风格选择 + 结构大纲

---

### 第四步：内容生产

子代理分工架构：

```
写作Agent ──→ 根据素材+大纲+规范生成初稿
                ↓
视觉Agent ──→ 根据内容生成SVG/图表/配图
                ↓
格式Agent ──→ 插入互动引导/往期精选/作者信息
```

**规范知识库**（技能知识库组件）：

| 规范 | 适用范围 | 内容 |
|------|----------|------|
| 公众号运营铁律 | 全部文章 | 标题公式/互动引导/往期精选/作者信息 |
| KIWI日记铁律 | 仅KIWI日记 | 角色定位/称谓一致/中英文切换/情感调性 |
| 飞书文档规则 | 仅飞书文档 | 不加Kiwi区块/按指令创建 |
| 通用写作规则 | 全部 | 纯文字超300字插表格/可视化/无模板痕迹 |

**组件映射**：子代理分工 + 技能知识库
**输出**：完整文章初稿（含配图）

---

### 第五步：审核校验

AI自检清单：

```
□ 标题是否使用核心数据/事件 + "你"的关联 + 系列标识？
□ 正文是否无板块标签（无【今日钩子】/元描述/模板痕迹）？
□ 纯文字是否超过300字？→ 是否穿插了表格或可视化？
□ 文末是否有互动引导（转发+评论+在看）？
□ 文末是否有3篇往期精选？
□ 文末是否有作者信息（海风老师身份+微信）？
□ 如果是KIWI日记：是否过了8条角色一致性检查？
□ 如果是KIWI日记：是否加了Kiwi简介区块？
□ 如果不是KIWI日记：是否确认没加Kiwi简介区块？
□ 是否有任何引用了已更新/已删除的旧规则？
```

**安全阀**（古德哈特定律陷阱）：
校验规则**不允许AI自己修改**。发现问题只能暂停循环、请求人工介入。

**组件映射**：技能知识库（校验规则）
**输出**：审核通过（进入发布）/ 审核不通过+修改建议

---

### 第六步：发布交付

根据用户指令决定输出格式：

| 指令 | 输出 | 发布路径 |
|------|------|----------|
| "写一篇公众号文章" | 输出公众号排版HTML（互动引导+往期精选+作者信息） | **→ 直接复制到公众号后台发布** |
| "生成飞书文档" | 创建飞书文档 | → 飞书内部使用 |
| "保存" | 本地文件（MD/SVG/HTML） | → 存档 |

**公众号发布规范（新增）**：
- 输出格式：HTML文件（适配公众号阅读样式）
  - 字号：正文15-16px，标题18-20px
  - 字体：系统默认（微软雅黑/苹方）
  - 行高：1.75，段间距1.5em
  - 颜色：正文#333，标题#000
  - 图片居中，宽度自适应
  - 无外部CSS依赖（公众号后台过滤外部样式）
- 不需要用户手动调格式，复制到公众号后台即可发布
- 系列文章自动带上编号

**隔离工作树**：每篇文章独立目录，素材/草稿/终稿/配图互不干扰。

**组件映射**：隔离工作树 + 连接器（飞书）+ 公众号排版规范
**输出**：公众号可直接发布的内容

### 第七步：效果追踪（2026-06-16升级版）

**跟踪闭环流程（三步）：**

#### 7.1 数据采集
用户发布后1-2天，发公众号后台数据截图给我。
→ 调用 `screenshot-extract` Skill 用 easyocr 提取截图中的文字
→ 自动解析关键指标：阅读量、分享数、留言数、完成率、跳出位置、评论内容

#### 7.2 热点融合分析
提取数据后同时执行两个动作：

| 动作 | 工具 | 目的 |
|------|------|------|
| 分析上期数据 | Elite记忆检索 | 对比历史表现，发现趋势 |
| 搜索当前热点 | WebSearch | 捕捉AI/科技热点话题 |

分析输出（3个结论）：
1. **上一篇复盘** — 标题有效吗？内容节奏对吗？读者在哪里弃读？
2. **热点关联** — 当前热点和上期主题能联动吗？标题能否加热点关键词？
3. **调整建议** — 下期的标题方向、内容角度、风格选择的优化方案

#### 7.3 记忆沉淀+下期启动
将完整分析结果写入Elite记忆（L2向量库）：
```
{
  主题: "Day 166: IP白名单...",
  日期: "2026-06-16",
  阅读量: 800,  分享数: 30,  留言: 5,
  完成率: 63%,
  跳出位置: "40%-60%段落",
  热词: "IP白名单/Loop/具身智能",
  评论反馈: ["评论区问最多的是XXX"],
  调整建议: {
    标题: "强化IP白名单+热点关联",
    内容: "缩短引言段",
    角度: "从技术细节转向行业意义"
  }
}
```

**组件映射**：持久化记忆（Elite）+ 连接器（Screenshot-extract + WebSearch）
**输出**：复盘报告 → 直接进入下一轮LOOP的Step1素材收集

## 三、最小可行性方案（现在即可启动）

不需要自动化工具，先手动跑通闭环：

**第1步**：下次写公众号文章时，按7步流程走一遍，感受每个环节

**第2步**：把素材收集+内容生产合并——让AI一步完成（你现在已经在做）

**第3步**：写完让AI过一遍审核清单（刚整理的那10条）

**第4步**：发布后告诉AI"这篇发完了"，让AI把经验写回Elite记忆

跑顺手工闭环后再考虑自动化触发。

## 四、三个补件（完整内容见 `LOOP方案_补件_完整版.md`）

### 补件一：选题规则判断框架
素材→风格的决策树，覆盖KIWI日记/技术分析/趋势预判三种风格的选择逻辑。

### 补件二：统一审核清单
合并所有散落规则为26条可执行checklist + 5条封禁清单，分为通用项（#1-#10）和风格专项（#11-#26）。

### 补件三：Elite公众号文章记忆Schema
L2向量存储结构、记忆读取策略（素材收集时自动检索）、记忆写入触发点（文章发布后自动执行）。

## 五、后续可能的升级路径

- **自动触发**：配置automation，每天下午固定时间触发KIWI日记生成
- **多Agent审核**：写作Agent和审核Agent用不同模型
- **读者反馈闭环**：公众号评论数据回写Elite，影响选题决策
- **模板化**：按风格建立内容模板（KIWI日记模板/技术分析模板/趋势预判模板）
