# 第一部分：咨询工作流（五阶段咨询架构）

整体咨询流程分为五个阶段，每个阶段有明确的进入/退出标准、交付物和审批节点。

```
        Phase 0          Phase 1          Phase 2          Phase 3          Phase 4
    ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
    │  商机与签约   │→│  诊断与基线   │→│  策略设计   │→│  实施落地   │→│  持续运营   │
    │  (1-2周)    │ │  (2-3周)    │ │  (2-4周)    │ │  (4-8周)    │ │  (月度)     │
    └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘ └─────────────┘
           ↓              ↓              ↓              ↓              ↓
       GO/No-Go      GO/No-Go      GO/No-Go      GO/No-Go      GO/No-Go
```

## Phase 0：商机与签约（Lead-to-Win）

### 目标
从初步接触到签署合作，完成需求对齐和范围锁定。

### 工作流

```
客户接触
    ↓
初步需求访谈（30min，免费）
    ↓
GEO 成熟度快速评估（免费）
    ↓
提案书递交（Proposal）
    ↓
商务谈判
    ↓
签约 → 收取首期款
```

### 交付物

| 产出 | 格式 | 备注 |
|------|------|------|
| 客户需求访谈纪要 | 内部文档 | 记录痛点和期望 |
| GEO 成熟度快速评估 | PPT/PDF | 2-3 页洞察，展示专业度 |
| 咨询提案书（Proposal） | PPT | 含项目范围、方法论、团队、报价、时间线 |
| 服务合同（SOW） | 法律文件 | 含交付物清单、里程碑、付款条件 |

### 退出标准
- [ ] 双方签署服务合同
- [ ] 首期款到账
- [ ] 客户对接人/项目负责人确认
- [ ] Phase 1 数据接入准备启动

### GeoREADY 快速评估
签约前提供一份**轻量级 GEO 成熟度评估（30 分钟访谈后出）**，包含：

```
评估维度                    权重
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
内容资产现状                  25%
AI 搜索能见度（品牌+关键词）    30%
技术基础设施（Schema/结构化数据） 20%
内容生产能力                   15%
团队认知水平                   10%
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
总分 100 分，分为四级：
  L1-未起步 (0-30)    → 需要完整 GEO 建设
  L2-探索期 (31-55)   → 需要系统化方案
  L3-建设中 (56-80)   → 需要优化和规模化
  L4-领先者 (81-100)  → 需要持续监控和防御
```

## Phase 1：诊断与基线（Diagnosis）

### 目标
建立客户当前的 GEO 表现基线，识别核心差距，为策略设计提供数据基础。

### 工作流

```
项目启动会（Kickoff）
    ↓
全量数据采集（4 周）
    ┣━ AI 搜索能见度扫描
    ┣━ 内容资产盘点
    ┣━ 竞品 GEO 对比分析
    ┣━ 技术基础设施审计
    ┗━ 内部团队认知访谈
    ↓
基线报告（Current State Assessment）
    ↓
差距分析 → 优先级排序
    ↓
Phase 1 汇报与核准会
```

### 交付物

| 交付物 | 格式 | 内容 |
|--------|------|------|
| 项目启动材料（Kickoff Deck） | PPT | 项目目标、方法论、时间线、团队分工 |
| 当前状态评估报告（CSA） | PPT/PDF | 完整基线诊断，含数据面板 |
| GEO SCORE 基线评分卡 | Excel/飞书多维表格 | 7 大维度 × 客户评分 |
| 竞品 GEO 对标矩阵 | PPT | Top 5 竞品的 AI 搜索表现对比 |
| 差距分析矩阵（Gap Matrix） | PPT/Excel | 按影响/难度分层 |

### 交付物

| 交付物 | 频次 | 格式 |
|--------|------|------|
| Sprint 工作计划 | 双周 | Excel/项目管理工具 |
| 内容生产日历 | 周度 | 多维表格 |
| Schema 实施确认单 | 每次部署 | 检查清单 |
| 双周进展报告 | 双周 | PPT |
| 质量抽检报告 | 每次内容发布前 | 内部评分卡 |

### 质量门禁（Quality Gate）

每篇内容发布前，必须经过以下检查：

```
□ GEO 友好性检查
   - 标题是否包含核心关键词（天然语义，非堆砌）
   - 是否包含权威引用/数据/信源
   - 是否有 FAQ 或问答式结构
   - 是否有明确的实体标签（人物/公司/概念）

□ 技术合规检查
   - Schema.org Article/FAQ/Breadcrumb 是否正确
   - 页面加载速度达标
   - 移动端适配确认
   - 无爬虫屏蔽（robots.txt 未误拦）

□ 品牌一致性检查
   - 品牌信息准确
   - 链接指向正确
   - 联系方式/网站引用一致
```

### 退出标准
- [ ] 核心内容矩阵建设完成（根据合同约定的内容量）
- [ ] Schema 标签覆盖率达标（如：≥80% 的页面）
- [ ] AI 引用率有可量化的提升（基线对比）
- [ ] 客户方团队具备基本的独立运营能力
- [ ] 转入持续运营模式

## Phase 4：持续运营（Managed Operations）

### 目标
确保 GEO 表现持续稳定提升，应对 AI 模型更新和市场变化。

### 运营节奏

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐
│  周度运营    │    │  月度复盘    │    │  季度战略    │
│  (内容/监控) │    │  (数据/调整)  │    │  (方向/迭代)  │
└──────────────┘    └──────────────┘    └──────────────┘
       ↓                   ↓                   ↓
   运营级报告          执行级报告           战略级报告
```

### 交付物

| 交付物 | 频次 | 格式 | 备注 |
|--------|------|------|------|
| GEO 周报 | 每周 | 一页纸（PPT/PDF） | 本周关键变化 + 内容产出 + 异常提醒 |
| GEO 月度报告 | 每月 | PPT | 全维度复盘，含趋势分析和优化建议 |
| GEO 季度战略报告 | 每季度 | PPT | 行业趋势、策略迭代、ROI 复盘 |
| 实时监控面板 | 持续 | 飞书/DataV | AI 能见度实时数据 |

### 监控报警机制

```
GEO 监控三级报警
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔴 P0 — 紧急
  触发条件：品牌在 AI 回复中的提及率单周下降 ≥30%
  响应时间：4 小时内
  动作：根因分析 → 紧急内容调优 → 48 小时内恢复

🟡 P1 — 警告
  触发条件：核心关键词 AI 覆盖率下降 ≥15% 或竞品新内容超越
  响应时间：24 小时内
  动作：差距分析 → 内容更新计划

🔵 P2 — 关注
  触发条件：新 AI 模型上线/版本更新导致回复模式变化
  响应时间：3 个工作日内
  动作：适配性评估 → 策略微调
```

### 退出标准（合同到期/终止）
- [ ] 最终复盘报告交付
- [ ] 客户方独立运营能力交接完成
- [ ] 数据资产（基线/趋势/方法论）完整移交
- [ ] 过渡期支持安排明确

| schema_types | json | 使用的 Schema 类型列表 |
| word_count | int | 字数 |
| keywords | json | 核心关键词 |
| ai_citation_count | int | 被 AI 引用的次数 |
| last_updated | datetime | 最后更新时间 |

### 竞品对标表（competitor_benchmark）

| 字段 | 类型 | 说明 |
|------|------|------|
| benchmark_id | string | 对标 ID |
| client_id | string | 客户 ID |
| competitor_name | string | 竞品名称 |
| competitor_url | string | 竞品域名 |
| dimension | enum | AI可见度/内容质量/技术基建/权威性 |
| client_score | float | 客户得分 |
| competitor_score | float | 竞品得分 |
| gap | float | 差距（负值=落后） |
| snapshot_date | datetime | 采集时间 |

## 2.3 GEO SCORE 评分算法

### 评分公式

```
GEO SCORE = Σ(W_i × S_i) / Σ(W_i) × 100

其中：
  W_i = 第 i 个维度的权重
  S_i = 第 i 个维度的归一化得分（0-1）
```

### 权重配置（默认版）

| 维度 | 权重 W_i | 说明 |
|------|---------|------|
| AI 可见度 | 30% | 最核心的输出指标 |
| 内容权威性 | 20% | AI 模型选择内容的标准 |
| 技术基建 | 20% | Schema/语义结构化是基础门槛 |
| 内容质量 | 15% | 影响引用深度 |
| 品牌信号 | 10% | 长期信任积累 |
| 结构化知识 | 5% | 增量优势项 |

### 子项评分规则（示例：AI 可见度）

```
AI 可见度评分 S_visible = 0.4 × 品牌提及率 + 0.3 × 覆盖率 + 0.2 × 首屏占比 + 0.1 × 引用正面度

其中：
  - 品牌提及率 = 提及品牌的查询数 / 总查询数
  - 覆盖率 = 品牌出现的 AI 平台数 / 总平台数
  - 首屏占比 = 排在第一的查询数 / 总提及查询数
  - 引用正面度 = AI 中品牌内容正向/中性回复比例
```

## 3.1 战略级报告模板（Tier 1）

### 结构

```
Page 1: 封面
  标题：[月份/季度] GEO 表现回顾
  副标题：[客户名称] × [咨询方]
  日期

Page 2: 执行摘要（Executive Summary）
  一句话结论：GEO 表现 [提升/下降] X%，主要驱动因素/核心问题
  3 个关键数据点
  本期重点事项

Page 3: GEO SCORE 总览
  趋势图（月度 GEO SCORE 变化曲线）
  当前分数 vs 基线 vs 目标
  7 维度雷达图对比（当前 vs 基线）

Page 4-5: AI 能见度分析
  各平台可见度对比柱状图
  核心关键词覆盖率矩阵
  提及内容 Top 10

Page 6-7: 竞品对标
  竞品得分对比（雷达图）
  差距变化趋势（月度时间序列）
  竞品新出现内容/策略预警

Page 8-9: 内容效果分析
  每篇内容的 AI 引用率
  高绩效内容特征提炼
  内容类型分布 vs 效果分布

Page 10-11: ROI 回顾
  实际产出 vs 计划
  流量/引用/品牌声量变化与投入对比
  预测 vs 实际趋势线

Page 12-13: 下期策略建议
  值得加大投入的领域
  需要调整的方向
  下季度目标建议

Page 14: 附录
  方法论说明
  数据源说明
  术语表
```

### 呈现原则（McKinsey 标准）

```
1. 结论先行（Pyramid Principle）
   - 每一页第一句话就是这页的结论
   - 后面的数据/图表是对结论的支撑

2. MECE 分解
   - 每个分析维度互斥且穷尽
   - "其他"类别不得超过总分析篇幅的 10%

3. 数据可视化
   - 一个图表只讲一个故事
   - 颜色编码一致：绿色=达标，黄色=警告，红色=未达标
   - 避免饼图，优先使用条形图/线图/瀑布图

4. 标题即结论
   - ❌ "AI 可见度分析"
   - ✅ "Q3 核心关键词 AI 引用率提升 25%，首屏占比达 40%"

5. 1-3-5 原则
   - 每页不超过 1 个核心观点
   - 不超过 3 个支撑论据
   - 不超过 5 个数据点
```

# 第四部分：交付标准与质量体系

## 4.1 交付物质量标准（McKinsey 级）

| 维度 | 标准 | 检查方式 |
|------|------|---------|
| **结构力** | 每个交付物有清晰的叙事线；结论先行 | 内部审阅 |
| **数据严谨** | 所有数据标注来源、采集时间、精度 | 数据溯源检查 |
| **视觉一致性** | 统一色板、字体、图标体系、布局模板 | 模板化 |
| **零错误** | 数据无计算错误，无错别字，无断裂链接 | 双人校验 |
| **交付时效** | 按 SOW 约定时间 ±1 个工作日内 | 项目管理系统追踪 |

## 4.2 内部审阅流程

```
初稿完成
    ↓（内部）
第一轮审阅（Peer Review）
    ┣━ 数据准确性核对
    ┣━ 逻辑完整性检查
    ┣━ 拼写/格式/排版
    ↓
修改稿
    ↓（内部）
第二轮审阅（Manager Review）
    ┣━ 战略层面：叙事线是否成立
    ┣━ 执行层面：建议是否可落地
    ┣━ 风险层面：有无遗漏风险
    ↓
终稿定版
    ↓（客户）
交付 → 确认 → 归档
```

## 4.3 术语表（Glossary）

| 术语 | 英文 | 定义 |
|------|------|------|
| GEO | Generative Engine Optimization | 为 AI 生成引擎优化内容，使其在 AI 回复中被引用 |
| AI 可见度 | AI Visibility | 品牌/内容在 AI 模型回复中出现的频率和位置 |
| AI 引用率 | AI Citation Rate | 品牌被 AI 回答引用的查询数占总查询的比例 |
| Schema 标签 | Structured Data Markup | 使用 schema.org 标准标记网页结构 |

| 品牌信号 | Brand Signals | 品牌在互联网上留下的可被 AI 识别的可信信号 |
| GEO SCORE | — | 综合衡量 GEO 表现的多维度评分 |
| MECE | Mutually Exclusive, Collectively Exhaustive | 互斥且穷尽的分解原则 |
| Pyramid Principle | — | 结论先行的沟通结构 |
