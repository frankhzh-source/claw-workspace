## 一、你的方案评审

你提的工作流骨架是正确的，但在生产环境中跑会有几个洞。我以三层身份逐层补充。

### 你已有的

```
① 拆解 → ② 关键帧 → ③ 图生图替换 → ④ 音频处理 → ⑤ 合成
```

### 漏了什么

三层视角交叉检出的缺失点：

```
视频剪辑大师视角
  ├─ 缺：剪辑节奏解构（每个镜头的精确时长和切点）
  ├─ 缺：转场分析（不仅是切点，还有转场类型/时长/缓动曲线）
  ├─ 缺：运动曲线提取（镜头推拉摇移的缓入缓出参数）
  ├─ 缺：色彩分级提取（不是"提取色调"这么简单，是每个场景独立的LUT）
  └─ 缺：声画同步点（音效与画面的对应关系——这是爆款的精髓之一）

AI FDE 视角
  ├─ 缺：一致性控制策略（关键帧之间的人物/场景外观如何保证不漂移）
  ├─ 缺：光流对齐（替换人物后，新人物身上的光照要匹配原场景的光照方向）
  ├─ 缺：背景补全（抠掉原人物后，背景空洞如何修复）
  ├─ 缺：分辨率梯队（不同阶段用不同分辨率——预览->精修->输出）
  └─ 缺：种子锁定策略（每帧用固定seed减少闪烁）

AI 架构师视角
  ├─ 缺：管线状态机（每步完成后有明确的验收标准才进下一步）
  ├─ 缺：失败回退机制（某步生成失败了，怎么回退重试而不重跑全量）
  ├─ 缺：资产树规范（帧/关键帧/条件图/音频/中间产物——文件结构怎么组织）
  ├─ 缺：并行化空间（哪些步骤可以并行跑）
  └─ 缺：增量能力（上次跑过的不重跑，断点续传）
```

```

│                        爆款视频复刻全管线                                 │

  ├─ Step 0.2 运动分析    → 每个镜头的 camera motion 曲线 + 主体 motion path
  ├─ Step 0.3 调色分析    → 每个场景的色板/LUT 提取
  └─ Step 0.4 音频分析    → BGM 结构 + 人声区间 + 音效时间点 + 声画同步点
      ↓

  └─ Step 1.2 音频拆解    → 人声轨 / BGM / 音效 分离（demucs / vocal remover）
      ↓

  ├─ Step 2.1 人物替换    → 参考图 → IP-Adapter/InstantID → 每场景多角度生成
  ├─ Step 2.2 场景替换    → 原场景背景补全 → 新场景生成 → 前后景合成
  ├─ Step 2.3 光照匹配    → 根据原场景光照方向/强度，校正新元素的 shading
  └─ Step 2.4 风格统一    → 调色 / 颗粒 / 景深匹配
      ↓
Phase 3：音频制作
  ├─ Step 3.1 人声替换    → 配音 / TTS / 原声保留
  ├─ Step 3.2 BGM 替换    → 选曲 + 对齐原 BGM 的结构
  └─ Step 3.3 音效移植    → 原音效时间点不变，替换音色或保留
      ↓
Phase 4：合成
  ├─ Step 4.1 画面合成    → 新画面 + 原结构时间线
  ├─ Step 4.2 音频混音    → 人声 + BGM + 音效 混音+母带
  ├─ Step 4.3 定版调色    → 全片统一色板
  └─ Step 4.4 输出         → 目标平台格式参数（抖音竖屏 / B站横屏 / 视频号）
```

### 2.2 Phase 0：分析（你漏掉的最重要阶段）

这是你方案里缺失最严重的环节。没有 Phase 0，后面的"复刻"只是凭感觉做。

**Step 0.1：节奏分析**

```
原视频
  │
  ├─ FFmpeg scene detect → 输出所有切点
  │   ffmpeg -i source.mp4 -filter:v "select='gt(scene,0.3)',showinfo" -vsync v2 frames/%05d.png
  │   输出：镜头1 (0-2.3s) | 镜头2 (2.3-5.1s) | 镜头3 (5.1-8.7s) ...
  │
  ├─ 多模态 AI 分析每个镜头的类型
  │   "镜头1：特写-产品展示-固定机位"
  │   "镜头2：中景-人物说话-缓慢推进"
  │   "镜头3：远景-场景展示-从左到右摇"
  │
  └─ 输出：剪辑节奏表
       场景编号 | 起止时间 | 时长 | 景别 | 镜头类型 | 运动描述
       ─────────────────────────────────────────────────────────
       1        | 0-2.3s  | 2.3s | 特写 | 固定    | 产品展示
       2        | 2.3-5.1 | 2.8s | 中景 | 缓慢推  | 人物说话
       ...
```

**Step 0.2：运动分析**

```
每个镜头的 camera motion 要量化，不能靠感觉：

  ├─ 镜头固定 → 参数: is_static=true
  ├─ 镜头推进 → 参数: zoom_start=1.0, zoom_end=1.3, ease=in_out
  ├─ 镜头平移 → 参数: pan_start=(x1,y1), pan_end=(x2,y2), ease=linear
  ├─ 镜头跟拍 → 参数: track_object=true, object_bbox=每帧坐标
  └─ 手持晃动 → 参数: shake_intensity=0.3, shake_frequency=中

提取方法：用 OpenCV 的光流分析 + 关键点追踪
```

**Step 0.3：调色分析**

```
每个场景提取独立 LUT（Look-Up Table）：

  ├─ 主色调 → hex: #2B5EA7（色相/饱和度/亮度）
  ├─ 对比度曲线 → 阴影 / 中间调 / 高光的函数曲线
  ├─ 色温 → 5000K / 7000K / 自定义
  └─ 颗粒/噪点 → 强度+大小

复刻时：新画面先调色对齐原场景的 LUT
```

**Step 0.4：音频分析**

```
音频是爆款视频的隐藏骨架，往往比画面更重要。

  ├─ BGM 结构分析
  │   前奏(0-3s) → 主歌(3-15s) → 副歌(15-25s) → 结尾(25-30s)
  │   BPM 检测: 128bpm
  │   关键节奏点: 每个鼓点的精确时间
  │
  ├─ 人声分析
  │   说话区间: 0:03-0:05, 0:08-0:12 ...
  │   语速: 4.2 字/秒
  │   情绪: 鼓励/讲解/惊讶
  │
  └─ 音效分析
      每个音效的时间点 + 类型（转场whoosh/强调ding/情绪音效）
      这是最容易被忽略但影响最大的细节
```

### 2.3 Phase 1：拆解（执行细节）

**Step 1.1：按场景分组的画面拆解**

```
原视频
    │
    ├─ FFmpeg 逐帧导出
    │   ffmpeg -i source.mp4 -q:v 2 source_frames/%05d.png
    │
    ├─ 按 Phase 0 的节奏表分场景
    │   scene_001/ → 镜头1的所有帧（00001-00069.png，共2.3秒）
    │   scene_002/ → 镜头2的所有帧（00070-00153.png，共2.8秒）
    │   scene_003/ → 镜头3的所有帧
    │   ...
    │
    ├─ 每个场景选关键帧（不是均匀抽帧，是选"特征帧"）
    │   第1帧（构图启动画面）
    │   中间帧（运动过程中的标志性姿态）
    │   最后帧（构图结束画面）
    │   中间如果有人物动作变化 → 再加1-2帧
    │
    └─ 特写帧（替换用的参考帧）
        每个场景选1帧最清晰的人物/产品正脸照
        → 用途：作为图生图的参考图
```

**一个 15 秒爆款视频的拆解产物：**

```
source_frames/          → 450 帧（30fps）
scene_001/              → 69帧（2.3s，产品展示特写）
  ├─ keyframe_001.png   → 起始构图
  ├─ keyframe_002.png   → 中间角度
  └─ ref_person.png     → 如果有人物的话
scene_002/              → 84帧（2.8s，人物说话中景）
  ├─ keyframe_001.png   → 起始
  ├─ keyframe_002.png   → 中间
  ├─ keyframe_003.png   → 结束
  ├─ ref_person.png     → 人物正脸参考
  └─ cond_pose.png      → 骨架图（人物姿势锁定条件）
scene_003/              → 108帧（3.6s，产品使用场景）
  ├─ keyframe_001.png   → 关键动作
  ├─ keyframe_002.png   → 关键动作
  ├─ ref_product.png    → 产品参考图
  └─ cond_depth.png     → 场景深度图
```

### 2.4 Phase 2：资产准备（关键帧生成的最佳实践）

**Step 2.1：人物替换策略**

```
不是"一张参考图生所有角度"，这是 AI 最容易翻车的地方。

正确做法：
  ┌──────────────────────────────────────────────────┐
  │ 场景A（正面特写）                                │
  │   ref=人物正面照                                  │
  │   → IP-Adapter FaceID + ControlNet Pose          │
  │   → 生成：保持正面长相 + 对齐原场景的姿势         │
  ├──────────────────────────────────────────────────┤
  │ 场景B（侧面中景）                                │
  │   ref=人物侧面照（如果只有正面照 → 先转角度生成） │
  │   → IP-Adapter + ControlNet Pose                  │
  │   → 生成：保持侧脸特征 + 对齐姿势                 │
  ├──────────────────────────────────────────────────┤
  │ 场景C（全身远景）                                │
  │   ref=人物全身照                                  │
  │   → LoRA（如果是固定人物，跑一个 LoRA 最稳）     │
  │   → ControlNet Pose + Depth                       │
  └──────────────────────────────────────────────────┘

关键规则：每个场景独立参考，不要试图"一张图通吃全片"
```

**Step 2.2-2.3：场景替换 + 光照匹配**

```
场景替换有三层，不是"直接图生图"：

  L1 — 纯替换背景（最简单）
     原图 → 抠出主体 → 生成新背景 → 合成
     适合：绿幕感/主体清晰的画面

  L2 — 保留原场景结构，换视觉风格
     原场景 → ControlNet Canny 锁定构图 → 新 prompt 生成
     适合：换场景风格但保留空间结构

  L3 — 完全替换场景（包括主体）
     原场景 → 抠掉原人物 → 背景补全（inpaint）
            → 生成新人物 → 光照匹配（新人物阴影方向=原场景阴影方向）
            → 合成
     适合：需要彻底换内容

光照匹配是这里最容易被忽略的：
  原场景阴影方向：左前方 30° 顶光，强度中等
  新生成人物阴影方向必须对齐
  不匹配 → 一眼就是"P的"
```

### 2.5 Phase 3：音频制作（控制爆款密码）

```
音轨分离（demucs / vocal remover）：
  ├─ 人声轨 → 如果换人声 → TTS 生成新台词 / 真人配音
  │            如果不换人声 → 保留原轨
  ├─ BGM 轨 → 结构对齐
  │   原BGM的关键节点：第3秒进鼓、第15秒副歌高潮、第25秒淡出
  │   新BGM必须在这个时间点上对齐——这是"听起来像"的核心
  └─ 音效轨 → 保留时间点，可替换音色
      转场whoosh / 强调音 / 环境音
      这些时间点写在 Phase 0 的 audio_map.json 里
```

---

### 2.6 Phase 4：合成（收尾工程）

```
合成不是简单的"拼起来"：

  画面合成
  ├─ 时间线恢复 → 按 Phase 0 的节奏表，精确到帧对齐
  ├─ 转场恢复 → 硬切/淡入淡出/滑动 → 参数对齐原视频
  ├─ 调色→ 每场景用 Phase 0 提取的 LUT 统一调色
  └─ 画质输出 → 5090D 跑 Topaz 超分到 4K

  音频合成
  ├─ 人声对齐 → 配音的音量/空间感/混响匹配原场景
  ├─ BGM 对齐 → 新 BGM 在关键节点与原 BGM 保持一致
  └─ 音效对齐 → 原有时间点一个不落

  定版
  ├─ 全片统一检查（色温/音量/字幕/黑场）
  └─ 目标平台参数输出
```
