本文档经过三重推理排查后编制：

- **树形推理**：根节点（合作共赢+保护核心）→ 5 主枝（摸底/对齐/保护/合作/风险）→ 15 子节点 → 40+ 叶子节点（具体问题与策略），逐枝推演排除低价值路径

## 一、全局定位：你在这张牌桌上的位置

### 1.1 模型厂的困境

VIDU 来访不是来"教你怎么用 AI"——他们是来**找应用闭环的**。字节跳动旗下的视频生成模型面临的核心问题是：技术很强，但不知道电商客户真正要什么、愿意为什么付费、工作流怎么嵌入企业已有系统。

### 1.2 你的价值

你不是普通 API 用户。你是三个角色的复合体：

| 角色 | 对 VIDU 的价值 | 谈判杠杆 |
|------|---------------|---------|
| 电商 AI 工作流构建者 | 验证 VIDU 在真实场景的可嵌入性 | 中 |
| AI 落地培训师（触达企业决策者） | 分发渠道——他们很难直接触达企业端 | ⭐ 高 |
| GEO/行业顾问 | 理解视频内容在搜索生态的表现规律 | 中 |

**核心认知**：你是一个分发渠道 + 场景验证器，不是终端用户。你的谈判地位比你以为的高。

### 1.3 你的核心风险

模型厂拿到你的数据后可以：

1. 把你的 Prompt 模式做成默认模板 → 你的 Prompt 知识贬值
2. 把你的质量评估标准内化为模型优化方向 → 你的判断力资产贬值
3. 通过你的案例接触你的客户 → 你被"中间商化"
4. 用你的场景数据训练电商垂直模型 → 卖给所有电商客户，你的工作流被通用化

## 二、树形推演：五枝策略体系

### 第一枝 · 摸清对方（进场阶段）

**目的**：判断 VIDU 当前的真实成熟度，确认基础可用性。

#### 1A · 模型能力摸底

| 问题 | 意图 |
|------|------|
| 文生视频最高分辨率/时长？支持 4K 吗？ | 电商视频需要高清细节（纹理、标签文字） |
| 图生视频的一致性控制到什么程度？ | 给商品图→出展示视频，外观不能变 |
| 支持哪些镜头运动控制？ | 电商视频需要标准化运镜语言 |
| 多片段/多镜头生成长度上限？ | 15-30秒电商视频通常需多段拼接 |
| 视频风格控制粒度？ | 品牌调性一致性是客户刚需 |

#### 1B · API 技术规格

| 问题 | 意图 |
|------|------|
| 并发上限和延迟？ | 批量场景（100+ SKU/次）是否可行 |
| 定价模式？按时长/分辨率/token？ | 帮客户估算 ROI |
| 异步生成 + Webhook 回调？ | ⭐ 嵌入 Agent 工作流的必要条件 |
| 输出格式？MP4/逐帧序列帧？ | 后期剪辑需求 |
| 水印策略？企业版去水印方案？ | 客户品牌合规 |

#### 1C · 对方角色判断

| 来访者角色 | 特征 | 你的应对 |
|-----------|------|---------|
| 产品经理 | 问场景/需求/痛点 | ⚠️ 最危险——他最想拿走你的 Prompt。给抽象场景不给细节 |
| 技术/工程师 | 问集成细节 | 安全对话区——聊 API 规范、回调机制 |
| 商务/BD | 问客户量/预算 | ⚠️ 警惕——他可能想绕过你接触客户。模糊回答 |
| 高管/VP | 问行业判断 | 思想对等对话——建立信任关系 |

**树形排除**：如果来的是纯技术团队（无PM/商务），风险降低 50%，可以多聊技术细节。如果 PM+商务同时出现，应大幅收紧信息输出。

### 第二枝 · 工作流对齐（核心环节）

**目的**：评估 VIDU 的电商场景赋能能力。

#### 2A · 批量 · 一致性 · 可控性

**核心问题链**：

1. **"每天给 500-1000 个 SKU 生成视频，如何保证同一商品在不同 prompt 下视觉一致？"**
   - 追问：Seed 控制 / 参考图权重调节 / 风格锁定机制
   - 场景：一个保温杯从"旋转展示"变成"倒水演示"，杯体不能变

2. **"常见类目是否有专项优化？"**
   - 服装：衣料动态、色彩还原
   - 美妆：肤质细节、颜色精准
   - 3C：屏幕内容、光效
   - 食品：质感、蒸汽、沸腾

3. **"支持 LoRA 或类似微调？"**
   - 训练成本？数据要求？部署方式？

#### 2B · 工作流嵌入深度

**展示你的架构**（抽象层级）：
"工作流基于 Vite+ReactFlow 构建，AI Agent 编排节点——从商品数据采集 → 文案生成 → 视觉方案 → 视频生成 → 发布。VIDU 在哪个环节嵌入最合适？"

| 问题 | 为什么 |
|------|--------|
| 异步回调机制如何设计？ | 视频生成是阻塞节点，需要可靠完成通知 |
| 支持多 prompt 排队生成吗？队列管理？ | 批量场景的基础设施 |
| 能与飞书多维表格联动吗？ | 你正在用的一体化方案 |
| 视频资产管理 API（存储/版本/删除）？ | 素材管理工程化 |
| 失败重试和错误码体系？ | 企业级稳定性 |

#### 2C · 行业定制纵深

以培训师身份问：

- "电商/零售行业有标杆客户吗？怎么用的？踩过什么坑？"
- "电商客户最常提的三种需求是什么？最让你们头疼的？"
- "未来 3-6 个月 roadmap 上，电商场景的优先级？"
- "视频内容的 SEO/GEO 有考虑吗？——帧文本语义索引、视频摘要生成？"

**横向验证**：对标 RunwayML 和 Pika 的电商策略，大多数视频模型在电商场景的嵌入深度还不够。VIDU 如果也没有，那大家都在同一起跑线。

### 第三枝 · 保护核心资产（防御层）

**目的**：拿到需要的能力，但不让核心资产外流。

#### 3A · 技术层保护：代理层架构

```
你的工作流 → [代理层] → VIDU API
                ↓
          本地缓存数据库
```

代理层做 4 件事：

**① Prompt 转化**
真实的 Prompt 模板不直接暴露。Agent 层从多维度属性动态组装文本，发出去的 flat text 无法逆向还原模板结构。

**② 结果缓存**
每次生成优先查缓存（内容 hash + 参数 hash）。命中直接返回，不调 API。三重好处：减少暴露 + 降成本 + 降延迟。

**③ 多模型路由**
代理层做抽象接口，同时跑 3+ 家模型做路由。不写死 VIDU。
- **最大谈判筹码**："我同时跑 3 家，你的表现排第 X"
- **防锁定**：任何一家涨价或改政策，随时切

**④ 请求脱敏**
- 条形码/二维码/Logo/品牌名 → 自动模糊或替换
- 客户 ID → 随机 hash
- 商品分类 → 标准类目名

#### 3B · 会话防御：怎么说、不说什么

| 对方问 | 你说 | 不说 |
|-------|------|------|
| "Prompt 怎么写的？" | "LLM Agent 动态生成，基于商品多维度属性" | Prompt 模板结构 |
| "工作流怎么编排？" | "ReactFlow 可视化，分 5 阶段" | 阶段间判断逻辑、异常处理 |
| "客户主要是哪些？" | "服装/美妆/3C 赛道中型电商" | 客户名称和联系方式 |
| "质量判断标准？" | "综合还原度/美观度/一致性" | 权重和阈值 |
| "用了哪些模型替代？" | "组合使用，场景不同选不同" | 真实用量分布 |

#### 3C · 可共享 vs 不可共享

**✅ 放心共享（建立信任）**
- 电商行业使用场景描述
- 视频质量反馈（"背景太复杂，主体不突出"）
- 行业普遍痛点
- 竞品差距评估
- 脱敏后的测试截图
- 业务定位

**❌ 红线（绝不共享）**
- 客户名单和合同 → ⭐ 被绕过就归零
- 完整 Prompt 模板体系 → ⭐ 拿去做预设模板
- 工作流编排源码 → ⭐ 节点逻辑是决策方式
- 质量评估量化标准 → ⭐ 判断力比生成能力更稀缺
- 完整模型选型矩阵 → ⭐ 失去谈判杠杆
- 客户付费意愿和预算 → ⭐ 他们了解后可绕过你定价

**纵向验证**：回顾软件行业 30 年，被平台侵蚀的 SI/ISV 无一例外犯了同一个错误——把客户关系暴露给了平台。

### 第四枝 · 长期合作（进攻层）

**目的**：从一次拜访变成深度合作通道。

#### 4A · 谈判筹码：你手里有什么

| 筹码 | 换什么 |
|------|-------|
| 真实电商测试场景 | API 折扣/额度、优先技术支持 |
| 高价值产品反馈 | 内测资格、Roadmap 知情权 |
| 培训渠道触达企业客户 | 联合品牌、推荐返佣 |
| 竞品对比数据 | 技术支持的优先级 |
| 行业认知和判断 | 思想对等地位、深度合作邀约 |

**原则**：每给出一项价值，收回一项资源。不空手给任何东西。

#### 4B · 会后跟进（24小时内）

1. 写 Meeting Notes 发给对方——你写下来的才是"事实"
2. 评估对方投入度：
   - 主动提出后续对接？还是"保持联系"？
   - 给了技术文档/开发者群入口？
   - 介绍了 product roadmap？
3. 决定是否进入下一轮：
   - 客套 → 维持礼貌不投入
   - 技术深挖意愿 → 约技术对接
   - 提出"你们用我们 API，我们出案例" → 🟢 绿灯，但先澄清协议

#### 4C · 必须拿到的承诺

- 数据使用条款：API 调用数据不用于模型训练（至少你的数据单独标注）
- API 稳定性：SLA 标准、限流策略
- 技术对接通道：专属对接人
- Roadmap 对齐：电商方向下个版本计划

### 第五枝 · 风险排查与排除

#### 5A · 被绕过风险（最高危）

**场景**：VIDU 通过联合案例接触到你的客户，直接提供服务，你被踢出链条。

**排除措施**：
- 客户信息匿名化（代理层自动替换）
- 不要展示真实的飞书多维表格
- 演示时用虚构的 demo 数据

**纵向验证**：Shopify 只做平台不做应用时，ISV 是安全的。Shopify 开始做 POS/支付/Fulfillment 时，ISV 开始被侵蚀。模型厂的节奏也一样——他们未来几乎必然做应用层。

#### 5B · Prompt 反噬风险

**场景**：你深调过的 Prompt 被 VIDU 拿去作为官方预设模板，人人可用，你的 Prompt 知识贬值。

**排除措施**：
- 代理层 Prompt 转化（发出去的和你真实的不一样）
- 每次发送加入随机噪音（同义替换、结构重排）
- 不在会话中展示原始的 Prompt 结构

#### 5C · 单模型锁定风险

**场景**：深度适配 VIDU 后，切换成本高，VIDU 涨价或改政策。

**排除措施**：
- 代理层保持抽象接口，随时换模型
- 维护一个模型评估矩阵（价格/质量/延迟/稳定性）
- 至少保持 1 家竞品模型在测试中

**逆向倒推验证**：假设 1 年后 VIDU 突然涨价 300%。如果你有代理层 + 多模型路由，切换成本是多少？→ 只需换 API 端点和做质量对比。如果没有，等于被绑架。

## 三、三维推理排查审计

### 横向比对：三类平台关系的通用风险

| 关系类型 | 平台 | 中介 | 最终结局 |
|---------|------|-----|---------|
| 电商 SaaS | Shopify | App开发者 | 平台复制成功 App，中介价值压缩 |
| AI 平台 | OpenAI | API 客户 | API 数据不用于训练（条款保护），但使用模式被分析 |
| 内容平台 | TikTok | 创作者 | 算法决定流量分配，创作者依赖平台 |

**对你意味着**：
- 模型厂走向应用层是确定的，只是时间问题
- 你的防御不在"阻止它发生"，而在"它发生时你还有自己的生态位"

**你的独特防御**：模型厂能做 API、能做预设模板、能做行业方案，但他们做不了两件事：
1. **培训 + 咨询服务**——需要人对人的信任
2. **多模型选型顾问**——他们不会推荐竞品

这两件事就是你的长期护城河。

### 纵向比对：30 年中介价值演化

| 时代 | 中介角色 | 被侵蚀时点 | 教训 |
|-----|---------|-----------|------|
| 90s | SI（系统集成商） | ERP厂商直签大客户 | 客户关系暴露是致命错误 |
| 2010s | ISV（独立软件开发商） | 平台复制成功 ISV 功能 | 单一平台依赖是风险 |
| 2020s | AI Workflow Builder | 模型厂做应用层 | 数据飞轮在模型厂手中 |

**你的经验迁移**：
- ❌ 不暴露客户关系（90s 教训）
- ❌ 不单平台绑定（10s 教训）
- ✅ 把数据飞轮的控制权留在自己手里（20s 教训）

### 逆向倒推验证

**起点**：最坏结局（T+1年）
> VIDU 绕过了你，你的工作流变成了他们的"一键生成"，你的客户直接调用 VIDU。你失去了所有议价能力。

**反向推导**：是什么导致了这一切？
1. 你暴露了客户名单 → 他们直接联系了客户
2. 你暴露了完整 Prompt 体系 → 他们内化为预设模板
3. 你没有建代理层 → 他们完全掌握你的调用模式
4. 你没有维护竞品关系 → VIDU 知道你没有别的选择

**今天的行动清单**：
1. ✅ 代理层架构（Prompt 转化 + 缓存 + 多模型路由 + 脱敏）
2. ✅ 客户信息保护（所有 demo 用虚构数据）
3. ✅ Prompt 体系分层（发出去的 ≠ 真实模板）
4. ✅ 至少维护 1 家竞品关系
5. ✅ 会话中按防御策略应对

**树形排除**：以上 5 项任意一项缺失，最坏结局实现概率增加 20%。

## 四、会议实战指南

### 开场建议（前 5 分钟）

不要一上来就提具体问题。先用 1 分钟介绍你的业务定位：

> "我是做电商 AI 落地和 GEO 优化的顾问。目前服务几家中型电商，帮他们搭建从商品数据到内容生成到发布的自动化工作流。我认为视频生成是电商内容化的下一个增长点，所以今天想看看 VIDU 在这个场景下的适配度。"

**这句话同时完成了三件事**：
- 建立了"我是顾问/渠道"而非"我是用户"的定位
- 暗示了你有客户资源可以对接
- 把对话基调定为"我来评估你的可用性"

### 60 分钟时间盒

| 时间 | 环节 | 目标 | 风险提醒 |
|------|------|------|---------|
| 0-5' | 双方介绍 | 建立"我是评估者"的定位 | 不要说得太详细 |
| 5-15' | ⭐ 你展示自己的工作 | 控制叙事框架——讲场景、讲客户需求、讲工作流（抽象） | 不要展示工作流源码和真实数据 |
| 15-35' | L2 核心提问 | 评估 VIDU 的场景适配度 | "一致性"和"回调"是最关键的试金石 |
| 35-45' | 对方介绍 VIDU 方向 | 听 roadmap 作为判断是否投入的依据 | 注意是否有电商方向的明确布局 |
| 45-55' | 合作方向讨论 | 抛出合作意向，观察对方反应 | 不要主动承诺任何高价值资源 |
| 55-60' | 索要资源 + 收尾 | 要联系人、要文档、确认下一步 | 最后补一句："后续我们先做小范围测试验证"——降低承诺感 |

### 四个信号灯

| 信号 | 颜色 | 含义 | 你的动作 |
|------|------|------|---------|
| 对方主动给你 API 文档 | 🟢 | 认真合作 | 可以给更多测试反馈 |
| 对方问你的客户是谁 | 🔴 | 绕道意图 | 回答"行业通用描述"，不点名 |
| 对方愿意签协议保护数据 | 🟢 | 有诚意 | 可以讨论更深度的合作 |
| 对方不停地问"你们的 Prompt" | 🔴 | 最危险 | 抽象回答，然后快速转移话题 |
