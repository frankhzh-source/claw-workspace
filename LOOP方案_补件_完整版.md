# LOOP方案 — 三个补件

> 补充公众号文章LOOP方案的待补部分：选题规则判断框架、统一审核清单、Elite记忆Schema。

---

## 补件一：选题规则判断框架

### 输入 → 判定的决策树

```
素材/想法来了
    │
    ├─ 是当日/近期的AI热点事件？
    │   ├─ 是 → 走「技术分析」风格（定义+演进+结构+验证）
    │   │    例：Loop Engineering刚爆发 → 技术分析
    │   │
    │   ├─ 是，且你有独特的未来视角？
    │   │   └─ 走「KIWI日记」风格（软科幻+AI头条+Kiwi点评）
    │   │       例：端侧AI视频 + 2031穿越视角 → KIWI日记素材
    │   │
    │   └─ 否 → 继续判断
    │
    ├─ 是你自己的推演/判断/预测？
    │   ├─ 是，且有可验证的论据支撑？
    │   │   └─ 走「趋势预判」风格（推演+判断+行动建议）
    │   │       例：三层布局规划、2028拐点推演
    │   │
    │   └─ 否 → 继续判断
    │
    ├─ 是日常观察/工作笔记/阅读笔记？
    │   ├─ 是，且有读者价值（不只是个人记录）？
    │   │   └─ 走「技术分析」风格（简洁版）
    │   │       例：LoRA训练经验总结
    │   │
    │   └─ 否 → 不适合发公众号
    │
    └─ 以上都不是 → 问海风要不要写，以及什么风格
```

### 风格特征速查表

| 风格 | 核心框架 | 目标读者 | 字数参考 | 配图需求 |
|------|---------|---------|---------|---------|
| **KIWI日记** | 真实对话驱动 + AI头条表格 + Kiwi金句收尾 | 关注AI趋势/想有差异化视角的人 | 1500-2500字 | AI头条表格 + 1张SVG |
| **技术分析** | 概念定义 → 演进 → 结构拆解 → 验证 → 启示 | 技术从业者/想学东西的人 | 2000-3500字 | 2-3张SVG/表格 |
| **趋势预判** | 输入 → 推演 → 判断 → 行动建议 | 决策者/创业者/想提前布局的人 | 2000-3000字 | 时间轴SVG + 表格 |

### 否决条件（以下情况不写）
1. 素材来源单一且无法交叉验证
2. 差异化角度不足（市面上已有大量同质内容）
3. 和读者画像（AI落地/电商人）无关
4. 纯信息搬运，没有你的判断或视角

## 补件二：统一审核清单

> 合并所有规则为一份可逐条勾选的checklist。分为**通用项**和**风格专项**。

### 通用项（全部文章必过）

| # | 检查项 | 通过标准 | 来源 |
|---|--------|---------|------|
| 1 | 标题公式 | 使用「核心数据/事件 + "你"的关联」结构，非纯标题党 | 公众号运营铁律 |
| 2 | 正文纯净化 | 无板块标签（如【今日钩子】）、无元描述、无模板痕迹 | 用户偏好 |
| 3 | 视觉锚点 | 纯文字超过300字必须插表格或可视化 | 公众号运营铁律 |
| 4 | 互动引导 | 文末有转发引导 + 评论话题 + 在看提示 | 公众号运营铁律 |
| 5 | 往期精选 | 文末挂3篇往期文章链接 | 公众号运营铁律 |
| 6 | 作者信息 | 底部有「海风老师 | AI技术咨询 / LoRA模型训练 / 电商AI落地」+ 微信frankhzheng | 飞书文档/公众号 |
| 7 | 系列标识 | 如果是系列文章，标题带系列编号（Day XXX / 第XX篇） | 公众号运营铁律 |
| 8 | 事实准确性 | 引用的概念、数据、来源可交叉验证 | 审核校验 |
| 9 | 规则版本检查 | 确认没有引用已更新/已删除的旧规则 | 审核校验 |
| 10 | Kiwi区块判断 | 确认当前文章属于哪一类，判断是否需加Kiwi区块 | 2026-06-16修正 |

### 风格专项：KIWI日记（8条角色一致性检查）

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 11 | Kiwi对话定位 | Kiwi是思考者/提问者/翻译者，不是资料搬运工。不炫耀"我早就说了"，用类比/提问引导海风自己发现 |
| 12 | 海风自嘲边界 | 自嘲与事实一致，不歪曲当天实际工作。自嘲有分寸，不削弱Kiwi金句力量 |
| 13 | 中英文切换 | 仅在必要时切换英语，有过渡标记（`"..." Kiwi切换到英语模式，"..."`），英语句子有实质内容 |
| 14 | 人物称谓一致 | 同一文章不混用：对话中用"Jensen"，单独出现时用"黄仁勋"或"英伟达CEO" |
| 15 | AI特性保持 | Kiwi不出现"逐条念""翻页""拖进度条"等人类化动作，用"瞬间消化""像发现宝藏一样" |
| 16 | 情感调性 | "轻松+哲理"基调，结尾不突跳沉重话题。如有沉重感，必须前文铺垫+Kiwi觉察缓冲 |
| 17 | AI头条融入 | 融入当日AI头条新闻（表格承载），至少1条做Kiwi视角点评（不是复述，是关联/反差/洞察） |
| 18 | 穿越者设定 | Kiwi从2031年穿越回来，金句来源不是更聪明而是多知道5年。警告机制先隐后显 |

### 风格专项：技术分析

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 19 | 概念定义清晰 | 首段/第二段给出简洁定义，读者30秒内知道在说什么 |
| 20 | 结构有逻辑 | 有清晰的演进或拆解框架（定义→演进→结构→验证/启示） |
| 21 | 验证环节 | 关键结论有事实/数据/交叉验证支撑，非纯主观判断 |
| 22 | 避免术语堆砌 | 技术名词有解释或类比，外行也能看懂核心结论 |

### 风格专项：趋势预判

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 23 | 输入明确 | 开篇说明"基于什么信息/资料做出的判断" |
| 24 | 推演链完整 | 从输入到判断的逻辑链条清晰可追，不是"我觉得" |
| 25 | 行动建议具体 | 结尾给出可执行的行动建议（不限于"关注趋势"） |
| 26 | 时间线清晰 | 如果有时间节点（2026→2028→2031），标注清楚 |

### 封禁清单（永远不能做的）

| # | 禁止项 | 原因 |
|---|--------|------|
| X1 | Kiwi简介区块出现在非KIWI日记/早报文章中 | 2026-06-16修正规则 |
| X2 | 未经用户指令主动创建飞书文档 | 2026-06-16新增规则 |
| X3 | 建议类内容写成操作指南（"你可以这样做..."）而不直接做 | 用户偏好 |
| X4 | 文章中出现"我帮你总结一下""以下是详细分析"等AI模板痕迹 | 用户偏好 |
| X5 | SVG中使用CSS变量（var(--color-xxx)）导致渲染失败 | 2026-06-16故障复盘 |

## 补件三：Elite公众号文章记忆Schema

### 存储位置
- **L2向量库**（语义搜索）：文章核心内容和关键词
- **L1寄存器**（会话级）：当前文章生产过程中的状态
- **WAL日志**（防丢）：每一轮循环的步进记录

### L2记忆结构（推荐写入字段）

```json
{
  "layer": "L2",
  "type": "article",
  "content": "文章全文摘要（300字以内）",
  "metadata": {
    "title": "文章标题",
    "style": "KIWI日记 | 技术分析 | 趋势预判",
    "published_at": "2026-06-16",
    "series_day": null,           // KIWI日记编号
    "keywords": ["关键词1", "关键词2", "关键词3"],
    "sources": ["素材来源1", "素材来源2"],
    "tools_used": ["WebSearch", "Elite搜索", "飞书文档"],
    "issues_found": ["审核中发现的问题", "SVG修复"], 
    "feishu_url": "https://...",
    "iteration_count": 1,         // 这篇文章经过几轮LOOP优化
    "next_action": "下次写同类文章时注意X"
  }
}
```

### 记忆读取策略

在LOOP的**素材收集**和**选题决策**步骤中，自动查询Elite：
1. `elite search "公众号 近期选题"` → 最近3篇，避免重复
2. `elite search "KIWI日记 Day"` → 日记序列号自动递增
3. `elite search "往期精选 推荐"` → 自动获取3篇作为推荐链接（按时间倒序）

### 记忆写入触发点（LOOP第7步自动执行）

```
文章发布后 → 自动执行 elite remember "文章摘要" -c article \
  --metadata '{...}' 
```

也可通过MCP工具直接写入：`memory_remember` 或 `l2_add`。

## 使用方法

### 第5步审核时
1. 打开**统一审核清单**（补件二）
2. 先过通用项（#1-#10），确定是否加Kiwi区块
3. 按风格过专项（#11-#18 / #19-#22 / #23-#26）
4. 检查封禁清单（X1-X5）⚠️ 这些是绝对不能踩的
5. 审核不通过 → 返回第4步修改；审核通过 → 进入第6步发布

### 第3步选题时
1. 用**选题规则判断框架**（补件一）判断素材方向
2. 确定风格 → 自动匹配该风格的要求（字数、配图数、结构）
3. 输出大纲

### 第7步追踪时
1. 用**Elite记忆Schema**（补件三）写入
2. 下次循环的素材收集步骤会自动检索历史文章
