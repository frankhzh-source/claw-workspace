Tier 1 通用底座 — 所有场景都要用到的能力

1. Prompt Engineering (多模态)

这是文生图/文生视频/LLM 的核心控制面 — Prompt写不好, 工具再强也白费
  • 文生图Prompt: 主体+场景+风格+画质+负向词, 五要素法
  • LLM Prompt: 角色设定 + 任务拆解 + 示例引导 + 输出格式控制
  • 文生视频Prompt: 动作描述 + 镜头语言 + 时间轴控制
  • Prompt模板库搭建: 按场景分类, 持续迭代, A/B测试
  • System Prompt vs User Prompt 的职责分离
  • Few-shot / Chain-of-Thought / ReAct 模式在电商场景的适用性
核心控制面 你已有底子

2. AI生图工具链

从单张生成到批量生产, 从"玩"到"产能"的关键跃迁
  • FLUX/SD3.5: 最新基础模型, 出图质量天花板
  • ComfyUI: 节点化工作流, 批量生产 + 复杂管线 (换脸/换背景/ControlNet)
  • LoRA训练: 用品牌/产品图片训练专属LoRA, 保持风格一致
  • ControlNet/IP-Adapter: 精准控制构图/姿势/面部
  • 产品图专用管线: 白底图 → 场景融合 → 模特换装 → 批量套版
  • 质量评估: CLIP Score / Aesthetic Score / 人工抽检标准
FLUX ComfyUI LoRA

3. LLM 应用层能力

客服/运营/直播话术/选品分析 都靠这个
  • 知识库+RAG: 把商品库/FAQ/政策文档变成可问答的知识
  • 意图识别与路由: 用户说"能便宜点吗" → 路由到议价策略
  • 多轮对话管理: 上下文窗口管理 + 对话摘要 + 关键信息提取
  • 结构化输出: JSON Mode / Function Calling 做商品推荐/订单查询
  • 内容生成: 标题优化/详情页文案/评价回复/营销短信
  • 主流模型选型: GPT-4o / Claude / DeepSeek / Qwen 在各场景的取舍
RAG LLM 你已有底子

4. 数据处理与自动化

从"手工做一张图"到"自动做一千张图"的能力
  • 商品数据处理: CSV/Excel读写, 图片批量命名, 元数据管理
  • Python脚本: 批量调用API, 图片处理(Pillow/OpenCV), 流程自动化
  • API集成: 电商平台API (商品上传/订单查询/数据拉取)
  • Webhook/定时任务: 自动化触发 — 新品上架→自动生成素材
  • 成本控制脚本: API调用计数, Token用量监控, 成本预警
Python 你的强项
Tier 2 场景深化 — 六大业务场景的落地方法论

场景1: AI商品图/营销图生产

电商最刚需的AI场景 — 省掉模特拍摄/场景搭建/多SKU套版的巨额成本
知识要点
  • ▸ 电商图片分类: 主图/附图/详情页/A+页面/广告素材 — 不同用途不同规格
  • ▸ 白底图→场景图: 用ControlNet+Inpaint把白底商品放入生活场景
  • ▸ 模特换装: IP-Adapter面部保持 + 服装LoRA, 减少真人拍摄
  • ▸ 风格一致性: 品牌LoRA训练 — 用50-100张品牌素材训LoRA, 保持视觉统一
  • ▸ 多SKU套版: 一个Prompt模板 + 批量替换商品图 + ComfyUI自动化
  • ▸ 电商平台规范: 淘宝/京东/拼多多/TikTok Shop的主流图尺寸和审核规则
常见坑
  • ▸ AI生成的文字是乱码 → 预留文字位, 后期加字
  • ▸ 商品细节变形 → ControlNet Canny/Lineart 保形
  • ▸ 多图风格不统一 → 必须训LoRA, Seed固定只治标
  • ▸ 平台审核不通过 → 了解AI素材标注规则, 各平台不同
你已有积累 需要量产化能力

场景2: AI短视频/带货视频

信息流广告+商品短视频 — 抖音/快手/TikTok的素材需求无限大
知识要点
  • ▸ 工具选型: Runway Gen-3 / Pika / Kling(可灵) / Sora / 即梦 — 各自的优劣势
  • ▸ 视频Prompt工程: 镜头运动(推拉摇移)+主体动作+氛围+时长控制
  • ▸ 图生视频: 商品静态图 → AI动态展示视频, 适合电商场景
  • ▸ 视频后期组合: AI生成片段 + 剪映/CapCut拼接 + 配音 + 字幕
  • ▸ 短视频脚本结构: 钩子(3秒)+展示(10秒)+卖点(5秒)+CTA(3秒)
  • ▸ 平台适配: 抖音竖版9:16 / YouTube横版16:9 / 信息流规格要求
常见坑
  • ▸ 人物动作诡异 → 优先用图生视频(可控性>文生视频)
  • ▸ 画面闪烁/不一致 → 多看几遍, 人工筛选+多生成备选
  • ▸ 生成成本高 → Runway按秒计费, 需ROI核算
  • ▸ "AI感"太重 → 加滤镜+调色+真实素材混剪破局

场景3: AI智能客服

售前提升转化率, 售后降低人工成本 — ROI最直观的场景
知识要点
  • ▸ 意图分类体系: 售前咨询/议价/物流/退换货/投诉 — 每条意图对应策略
  • ▸ 知识库搭建: 商品信息/尺码表/售后政策/常见问题 — RAG检索增强
  • ▸ 多轮对话设计: 追问→澄清→确认→闭环, 不是一问一答
  • ▸ 情绪识别与安抚: 用户发怒时的降级话术和升级转人工机制
  • ▸ 订单系统对接: 查物流/查订单/发起退款 — API集成能力
  • ▸ 效果评估: 首解率/转人工率/用户满意度/响应时间
常见坑
  • ▸ "抱歉我是AI"死循环 → 别让AI承认自己是AI, 直接给答案
  • ▸ 知识库过时 → 商品变价/断货时必须实时同步
  • ▸ 乱承诺 → 价格/库存/时效的回复必须有API兜底, 不能靠LLM记忆

场景4: AI电商运营

从标题优化到数据复盘, 运营日常工作流全面AI化
知识要点
  • ▸ 标题优化: 关键词挖掘(生意参谋/抖音搜索) → AI生成 → A/B测试
  • ▸ 详情页: 卖点提取 → 结构化文案 → AI配图 → 套版生成
  • ▸ 评价管理: AI自动生成好评回复, 差评预警+人工介入
  • ▸ 数据复盘: 日报/周报自动生成, 异常检测(流量/转化率突变)
  • ▸ 选品分析: 竞品监控 → 趋势发现 → AI辅助选品建议
  • ▸ 营销内容: 短信/Push/社群文案批量生成+个性化
常见坑
  • ▸ AI标题堆砌关键词 → 要注入品牌调性和口语感
  • ▸ 数据复盘不结合业务 → AI不知道"昨天有活动所以流量高"
  • ▸ 选品分析要结合供应链 → AI只能给信号, 不能给决策

场景5: AI直播运营

话术生成+实时弹幕回复+数据监控 — AI做副驾, 真人做主播
知识要点
  • ▸ 排品策略: 引流款→利润款→福利款→逼单款, AI生成排品方案
  • ▸ 话术库: 开场话术/产品介绍/逼单话术/答谢话术/结束话术
  • ▸ 实时弹幕回复: 识别高频问题, 自动生成回复, 主播照读/助手代回
  • ▸ 数据监控: 在线人数/转化率/GPM实时预警 → AI给出调整建议
  • ▸ 直播复盘: 整场录播→AI分析→流量曲线/转化节点/话术效果
  • ▸ 直播间搭建: AI生成虚拟背景/场景素材
常见坑
  • ▸ 弹幕回复速度跟不上 → 弹幕量太大, 需要消息优先级过滤
  • ▸ AI话术太"模板化" → 需要注入主播个人风格
  • ▸ 实时数据延迟 → API接入+低延迟推送方案

场景6: 数字人主播

7x24小时直播, 覆盖长尾时段 — 从"能用"到"好用"还在路上
知识要点
  • ▸ 技术方案: HeyGen/D-ID/Synthesia 等SaaS, 或开源方案(MuseTalk/SadTalker)
  • ▸ 形象定制: 真人形象克隆(需录制素材) vs 虚拟形象(AI生成)
  • ▸ 声音克隆: ElevenLabs / GPT-SoVITS / CosyVoice — 中文效果对比
  • ▸ 智能讲解: 商品话术+数字人驱动+弹幕互动 三合一
  • ▸ 直播平台对接: 淘宝/抖音/视频号的数字人政策和限制
  • ▸ 效果评估: 停留时长/互动率/转化率 — 和真人主播对比
常见坑
  • ▸ "恐怖谷"效应 → 形象和动作不够自然, 观众排斥
  • ▸ 平台封禁风险 → 各平台对数字人政策不同, 需要提前确认
  • ▸ 口型不同步 → 声音和嘴型对齐是技术难点
  • ▸ 互动感差 → 纯脚本播放≠直播, 实时弹幕互动是关键
Tier 3 商业层 — 从"能做"到"能卖"

AI落地方案设计

怎么跟客户聊需求, 怎么出方案, 怎么报价
  • 需求诊断: 客户说"我想用AI" → 拆解到具体场景和指标
  • 方案结构: 现状分析→目标定义→技术方案→实施计划→ROI测算→风险预案
  • 技术选型决策树: 自建vs采购, 开源vs商业API, 通用模型vs领域微调
  • 报价模型: 按效果付费 / 按量付费 / 项目制 / SaaS订阅 — 哪种适合什么客户
  • 案例库积累: 每个落地方案都是一张名片

ROI测算与效果评估

客户/老板问的第一句话永远是"能省多少钱"或"能多赚多少"
  • 文生图ROI: 省掉多少拍摄费 = 模特费+摄影师+场地+后期 × 月SKU数
  • 客服ROI: 7x24覆盖+减少人工 = 减少X个人力 × 月薪
  • 视频ROI: 素材产能提升 = 月素材数 × 素材转化贡献
  • 隐性价值: 响应速度/客户体验/品牌调性一致 — 难以量化但必须提及
  • 效果评估指标: 转化率/满意度/成本/TAT(周转时间)/人工替代率

行业认知与趋势判断

知道AI电商的边界在哪里, 什么东西暂时做不了
  • 各平台AI政策: 淘宝/京东/抖音/拼多多/SHEIN/Temu对AI内容的态度
  • 竞争对手动态: 哪些品牌已经在用AI, 用到什么程度
  • 技术趋势: Sora/Veo对视频生产的冲击, GPT-5/Claude 5 对客服的升级
  • 成本趋势: API降价曲线, 开源模型追赶节奏
  • 边界判断: 什么时候AI是"锦上添花", 什么时候是"雪中送炭"

项目管理与交付能力

从"我会做"到"我能交付"的最后一步
  • 项目周期管理: PoC(2-4周) → MVP(1-2月) → 正式上线 → 持续优化
  • 效果验收标准: 上线前就定义好什么算"成功"
  • 客户培训: 教会客户/运营团队使用AI工具 (这是你已有的培训能力)
  • 持续优化: AI方案不是一次性的, 需要持续迭代Prompt/知识库/模型
  • 风险预案: API宕机怎么办? 效果变差怎么办? 平台政策变动怎么办?

和之前"深度学习知识体系"的关系:
那份是给"想自己做模型/微调/优化推理引擎"的人看的 — 那是算法工程师的路。
这份是给"用AI工具解决电商业务问题"的人看的 — 这是FDE/应用落地顾问的路。
两条路都需要, 但你是先走这条路, 用到瓶颈了再补深度的东西。

核心原则: 你的核心竞争力不是"会调参数", 而是"知道AI能在电商的哪个环节, 解决什么问题, 带来多少价值"。工具可以变, 这个判断力不会过时。
