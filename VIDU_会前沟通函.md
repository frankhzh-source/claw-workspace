# VIDU 会前沟通指南

## 一、要不要提前告知？—— 应该给

给，但给的是**框架**不是**试卷**。

好处：
- 让他们带对的人来（如果有技术问题，工程师要到场）
- 让他们带对的材料来（不会空手聊）
- 你控制了会议的叙事起点
- 姿态是"合作对齐"，不是"我来请教"

**不给的风险**：对方可能派纯商务来，技术问题答不上；或者准备了一堆你用不上的推广材料。

## 二、沟通函草稿（可直接发）

> **主题：关于下周三交流的准备建议**
>
> Hi [对方称呼]，
>
> 周三见面前，想提前对齐一下交流重点，这样双方准备更高效。
>
> **我的情况简要：**
> 我正在帮电商品牌搭建 AI 内容生成工作流，涵盖商品数据解析 → 文案生成 → 视觉方案 → 视频生成 → 多平台发布的全链路。目前主要用 ReactFlow 做可视化编排 + 飞书多维表格做数据流转。视频生成环节目前在测试多家模型，想看看 VIDU 在电商场景的适配度和嵌入深度。
>
> **希望了解的方向：**
>
> 1. **电商场景的模型能力** — 尤其是同一商品多角度/多场景生成时的一致性控制方案、批量生成的支持方式
> 2. **API 集成深度** — 异步回调/Webhook 机制、队列管理、与外部系统的联动能力
> 3. **行业专项支持** — 电商类目（服装/美妆/3C/食品）是否有专项优化或微调能力
> 4. **产品方向** — VIDU 在电商内容方向的近期规划和重点
>
> **如果方便，可以提前准备：**
> - 电商/零售相关的参考案例或场景说明
> - API 技术文档（特别是异步调用和回调部分）
> - 如果技术同学能一起参与更好，方便现场对齐集成方案
>
> 我们这边也会准备好具体的电商测试场景和反馈，争取见面高效产出。
>
> 周三见。

## 三、这封函的战术拆解

| 要素 | 写的什么 | 战术目的 |
|------|---------|---------|
| 第一段 | "提前对齐，高效准备" | 定调——这是工作对齐，不是闲聊 |
| "我的情况" | 工作流描述（抽象） | 提前建立定位——你不是普通用户 |
| "目前测试多家" | 一句话带过 | 暗示"我有竞品对比"，建立谈判杠杆 |
| 4个方向 | 具体但不暴露底牌 | 他们看了就知道带什么材料来 |
| "技术同学能参与更好" | 不动声色地要求技术人员到场 | 减少被纯商务打发走的风险 |
| "我们也会准备测试场景" | 双向承诺 | 姿态对等，不是单方面索取 |
| 结尾 | "高效产出" | 暗示"产出导向"的交流风格 |

## 四、如果你不想发完整邮件，也可以用这句话结尾

不想写太正式的话，可以在确认见面的消息里加一句：

> "对了，如果方便的话，带上 API 技术文档和电商场景的案例材料？我们这边也准备好具体的测试场景和 feedback，争取一次聊透。"

一句话同时做了四件事：要文档、要案例、暗示自己也在认真准备、表明"我不想浪费时间"。
